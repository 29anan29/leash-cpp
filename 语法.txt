本文件是 Leash 的**唯一权威设计稿**。编译器、标准库、文档均以此为准。
当前已实现子集见 AGENTS.md（"当前可实现的语言子集"），本稿中带 [规划]

升级总目标（对应本次任务四大块）：
  1) 原生 AI 原语：agent / tool / chain / rag / model 是语言级关键字，
     不是标准库模仿 LangChain 的 .ae 包。
  2) 安全模型：cap 能力类型 + requires 静态门控 + 最小特权 + 沙箱
     + 确定性 + 资源治理(fuel/timeout) + 调用审计，作为语言强制约束。
  3) 性能：字节码 VM + 编译期优化 + 检索/向量加速，给出量化目标。
  4) 先定设计（本稿），再分步实现，旧示例在过渡期内继续可用。


## 0. 安全模型（设计第一原则）
- 无环境权威：脚本默认不能接触文件/网络/时钟/进程；所有副作用都通过
  宿主注入的**能力(capability)句柄**完成，句柄不可伪造、不可凭空构造。
- 能力门控 I/O：out / in 是宿主注入的 IO 能力语法糖，不是全局函数；
  非持有 IO 能力的作用域内调用 out/in 编译报错。
- 最小特权：fn 用 requires 声明所需能力，编译器做**能力流(capability
  flow)**静态校验——能力只能在被声明持有它的作用域内使用，不能逃逸。
- 确定性：默认无墙上时钟、无隐式并发；RNG 由带种子的能力驱动；
  @deterministic 作用域内禁止一切非确定源。
- 资源治理：运行时按 fuel 限制步数、按配额限制内存、限制递归深度，
  支持 @timeout 超时熔断；Agent 推理循环受 max_steps 上限保护。
- Agent 闭环白名单：Agent 仅能调用显式列出的 tool，不能执行任意代码；
  LLM 调用走受控的 Model 能力并被审计，脚本无法绕过宿主策略。
- 不安全隔离：package python.*（白名单）在独立沙箱进程执行，标注 unsafe，
  必须显式授权，且其内部调用同样受 fuel + 内存配额限制。
- 审计：每次能力调用由运行时结构化记录（能力名、入参、结果、步数消耗）。

## 10. 能力与安全（核心）[规划=本次重点实现]
cap IO {                              // 定义能力接口
  print: fn(string) -> unit
  read_line: fn(string) -> string
}
fn main requires io: IO               // main 由宿主注入 io 能力
  out("hi")                           // = io.print("hi")
unsafe python.tk as py                // 白名单 Python，标注 unsafe，沙箱执行
  let r = py.eval("1+1")
// 能力为不可伪造句柄；编译器追踪能力在作用域内流动(capability flow)
// unsafe 块/导入必须在宿主策略中显式授权，否则拒绝加载
// 每次能力调用由运行时结构化审计(能力名、入参、结果、步数)
//
// 实现要点（编译器侧）：
//   - parser：识别 cap / requires / unsafe / @fuel / @timeout / @deterministic
//   - typecheck：维护"当前作用域可用能力集合"，requires 声明向集合注入，
//     能力字段访问(io.print)必须在该能力于集合内；否则报错。
//   - vm/host：能力句柄 = 宿主侧不透明对象；脚本侧只能持有引用，
//     无法伪造；每次宿主调用记审计日志。


## 11. Agent 原生原语（核心，语言内建而非标准库）
// 11.1 tool：定义 LLM 可调用工具，本体受能力门控（工具内部仍是沙箱代码）
tool search(query: string) requires net: NetCap -> Result<string, Err>
  net.http_get("https://api?q=" ++ query)

// 11.2 agent：内建关键字，声明一个 Agent（组合 模型 + 工具 + 系统提示）
agent Helper requires io: IOCap, net: NetCap
  model     "gpt-4o"                 // 通过 Model 能力调用 LLM（必须持有）
  system    "你是一个安全助手"         // 系统提示
  tools     [search]                 // 白名单：Agent 仅能调用显式列出的工具
  memory    off                      // 记忆由 Memory 能力控制，默认 off
  max_steps 8                        // 推理循环最大步数（防失控 / 防 DoS）
  run(query: string) -> Result<string, Err>
    respond(query)                   // 内建：触发一次 Agent 推理-工具循环

// 11.3 内建调用（仅在 agent.run 作用域内可用）
respond(prompt) -> Result<string, Err>   // 调用 LLM 并自动执行工具循环
observe(msg: string)                     // 向 Agent 上下文注入观察

// 11.4 chain：声明式编排多个步骤（类似 LangChain Runnable 序列）
chain summarize_then_translate(text: string) -> Result<string, Err>
  s = text |> summarize()
  s |> translate("zh")

// 11.5 rag：原生检索增强原语（本次升级新增）
rag MyKB requires vec: VecCap, model: ModelCap
  source  "./docs/*.md"              // 语料来源（受 file 能力门控）
  embed  "bow"                       // 嵌入方式：bow(词袋)/dense(模型)
  top_k   3
  run(q: string) -> Result<string, Err>
    ctx = retrieve(q)                // 内建：向量检索 top_k
    prompt = "依据资料回答:\n" ++ ctx ++ "\n问题:" ++ q
    respond(prompt)                  // 复用 agent 推理循环

// 11.6 model：多供应商/多模型原生切换（本次升级新增）
model openai   = "gpt-4o-mini"   @base "https://api.openai.com/v1"
model claude   = "claude-3-5-sonnet" @base "https://api.anthropic.com/v1"
model ollama   = "llama3"        @base "http://localhost:11434/v1"
model deepseek = "deepseek-chat"  @base "https://api.deepseek.com/v1"
// 切换：within agent，model = claude 即时切换；全局 model 变量可运行时重赋值。
// 所有 model 调用均经 ModelCap 能力，令牌上限/审计由宿主控制。

// Agent 安全要点：
//  - tools 是闭包白名单，LLM 无法调用列表之外的任何函数 / 代码。
//  - 每个 tool 本体运行在沙箱中，受 fuel + 内存配额限制。
//  - model 调用走 Model 能力：宿主控制模型、令牌上限、审计，脚本不可绕过。
//  - max_steps 限制推理循环次数；每一步扣 fuel，超时熔断。
//  - rag 的 source 受 file 能力门控，检索仅限授权路径。

## 15. 性能（本次升级目标）[规划]
设计原则：确定性优先的字节码 VM，不引入不可控的 JIT 不确定性。
- 编译器：
  * 单遍/两遍前端；常量折叠、死代码消除、尾调用复用栈帧。
  * 字节码：紧凑 opcode + 常量池；函数内联小工具调用。
  * 编译期能力流分析（§10）与类型推断，减少运行时检查。
- 虚拟机：
  * 基于栈的字节码 VM，指令分派用计算跳转（computed goto）降低开销。
  * 值采用 tagged union + 小对象内联，减少堆分配。
  * fuel 计数为 VM 主循环每步自减，超时用单调时钟采样（不破坏确定性）。
- 检索/向量：
  * 向量库支持**构建期索引**（IVF/flat 量化），query 走 top-k 近似/精确。
  * 词袋嵌入走预计算词表 + 稀疏向量点积；dense 走宿主模型能力。
  * 大语料 rag.source 做分块缓存，避免每次查询重算嵌入。
- 量化目标（CI 基准，2.0 GHz 单核）：
  * 空 fn main 启动到首行 out：< 5 ms（含 assemble + 编译 + VM 启动）。
  * 100 万次整数加法循环：< 30 ms（fuel 放开时）。
  * 1 万文档语料的 top-5 检索：< 20 ms（索引已构建）。
  * chat(mock) 往返：< 1 ms（无网络）。

## 17. 完整示例（Agent 编排 + RAG + 多模型，原生写法）
package agent
cap NetCap { http_get: fn(string) -> Result<string, Err> }
cap IOCap { print: fn(string) -> unit }
cap ModelCap { chat: fn(string) -> Result<string, Err> }

model openai = "gpt-4o-mini" @base "https://api.openai.com/v1"

tool wiki(q: string) requires net: NetCap -> Result<string, Err>
  net.http_get("https://en.wikipedia.org?search=" ++ q)

rag Docs requires vec: VecCap, file: FileCap, model: ModelCap
  source "./kb/*.md"
  embed  "bow"
  top_k  3
  run(q: string) -> Result<string, Err>
    ctx = retrieve(q)
    respond("依据资料回答:\n" ++ ctx ++ "\n问题:" ++ q)

agent Assistant requires io: IOCap, net: NetCap, model: ModelCap
  model   openai
  system  "用简洁中文回答"
  tools   [wiki]
  max_steps 6
  run(q: string) -> Result<string, Err>
    respond(q)

fn main requires io: IOCap, net: NetCap, model: ModelCap
  let mut int:c = in("你的名字？")     // in = io 语法糖
  out("hello {c}")                    // out = io 语法糖
  match Assistant.run("Leash 是什么")
    | Ok r  -> out(r)
    | Err e -> out("failed")
  match Docs.run("RAG 的核心组件?")
    | Ok r  -> out(r)
    | Err e -> out("kb failed")
