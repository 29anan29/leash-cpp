

========================================================================
Aegis 语法规范（补全版）
========================================================================

## 0. 安全模型（设计原则）
- 无环境权威：脚本默认不能接触文件/网络/时钟/进程；副作用全部通过入口注入的能力(capability)完成。
- 能力门控 I/O：out / in 是宿主注入的 IO 能力的语法糖，不是全局函数。
- 最小特权：fn 用 requires 声明所需能力，编译器静态校验作用域内是否持有。
- 确定性：默认无墙上时钟、无隐式并发；RNG 由带种子的能力驱动。
- 资源治理：运行时按 fuel 限制步数、按配额限制内存、限制递归深度，可超时熔断。
- Agent 闭环白名单：Agent 仅能调用显式列出的 tool，不能执行任意代码；LLM 调用走受控的 Model 能力并审计。
- 不安全隔离：package python.*（白名单）在独立沙箱进程执行，标注 unsafe，需显式授权。

## 1. 保留关键字
控制/声明：fn let mut if else match for while loop return break continue
类型：type int float bool string unit cap requires
模块：package import
逻辑/值：true false and or not in out
Agent 原语（内建）：agent tool chain model system tools memory max_steps respond observe
修饰：unsafe  @fuel @timeout @deterministic

## 7. 函数

fn add(a: int, b: int) -> int

fn run(agent: AgentCap, path: string) requires agent
  let mut int:c = 12
  c = in("How old are you")         // in = IO 能力语法糖
  out("you will {c + 1} years old") // out = IO 能力语法糖
  agent.read_file(path)

## 10. 能力与安全（核心）
cap IO {                              // 定义能力接口
  print: fn(string) -> unit
  read_line: fn(string) -> string
}
fn main requires io: IO               // main 由宿主注入 io 能力
  out("hi")                           // = io.print("hi")
unsafe python.tk as py                // 白名单 Python，标注 unsafe，沙箱执行
  let r = py.eval("1+1")
// 能力为不可伪造句柄；编译器追踪能力在作用域内流动(capability flow)
// unsafe 块/导入必须在宿主策略中显式授权，否则拒绝加载
// 每次能力调用由运行时结构化审计(能力名、入参、结果)

## 11. Agent 内建原语（核心，类似 LangChain）
// 11.1 tool：定义 LLM 可调用工具，本体受能力门控（工具内部仍是沙箱代码）
tool search(query: string) requires net: NetCap -> Result<string, Err>
  net.http_get("https://api?q=" ++ query)

// 11.2 agent：内建关键字，声明一个 Agent（组合 模型 + 工具 + 系统提示）
agent Helper requires io: IOCap, net: NetCap
  model     "gpt-4o"                 // 通过 Model 能力调用 LLM（必须持有）
  system    "你是一个安全助手"         // 系统提示
  tools     [search]                 // 白名单：Agent 仅能调用显式列出的工具
  memory    off                      // 记忆由 Memory 能力控制，默认 off
  max_steps 8                        // 推理循环最大步数（防失控 / 防 DoS）
  run(query: string) -> Result<string, Err>
    respond(query)                   // 内建：触发一次 Agent 推理-工具循环

// 11.3 内建调用（仅在 agent.run 作用域内可用）
respond(prompt) -> Result<string, Err>   // 调用 LLM 并自动执行工具循环
observe(msg: string)                     // 向 Agent 上下文注入观察

// 11.4 chain：声明式编排多个步骤（类似 LangChain Runnable 序列）
chain summarize_then_translate(text: string) -> Result<string, Err>
  s = text |> summarize()
  s |> translate("zh")

// Agent 安全要点：
//  - tools 是闭包白名单，LLM 无法调用列表之外的任何函数 / 代码。
//  - 每个 tool 本体运行在沙箱中，受 fuel + 内存配额限制。
//  - model 调用走 Model 能力：宿主控制模型、令牌上限、审计，脚本不可绕过。
//  - max_steps 限制推理循环次数；每一步扣 fuel，超时熔断。

## 12. 错误处理
let r = agent.read_file(p)
match r
  | Ok s  -> out(s)
  | Err e -> out("error")
r ?? default                          // 提供默认值
do_thing()?                           // ? 提前返回 Err

## 14. 资源与确定性
@fuel(1_000_000)                      // 声明最大步数
@timeout(5s)                          // 超时熔断
@deterministic                        // 禁止非确定源(无种子时钟/RNG 报错)
// 运行时强制内存配额与最大递归深度

## 15. 完整示例（Agent 编排）
package agent
cap NetCap { http_get: fn(string) -> Result<string, Err> }
cap IOCap { print: fn(string) -> unit }

tool wiki(q: string) requires net: NetCap -> Result<string, Err>
  net.http_get("https://en.wikipedia.org?search=" ++ q)

agent Assistant requires io: IOCap, net: NetCap
  model     "gpt-4o"
  system    "用简洁中文回答"
  tools     [wiki]
  max_steps 6
  run(q: string) -> Result<string, Err>
    respond(q)

fn main requires io: IOCap, net: NetCap
  let mut int:c = in("你的名字？")     // in = io 语法糖
  out("hello {c}")                    // out = io 语法糖
  match Assistant.run("Aegis 是什么")
    | Ok r  -> out(r)
    | Err e -> out("failed")
