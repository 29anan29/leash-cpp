# 第1章 前言

## 1.1 什么是 Aegis

Aegis 是一门专为 AI Agent（智能体）开发设计的编程语言。它的名字来源于希腊神话中的"神盾"埃癸斯，象征着保护与安全。与传统的通用编程语言不同，Aegis 从诞生之初就将**安全性**和**可控性**作为核心设计目标。

在 AI Agent 快速发展的今天，我们面临一个根本性的挑战：如何让 AI 能够自主执行代码，同时又不会给系统带来安全风险？传统的语言要么过于开放（如 Python 的 `eval`），要么过于封闭，无法满足 AI Agent 灵活调用资源的需求。

Aegis 的答案是**能力安全（Capability Security）**模型。在这个模型中，程序没有任何默认权限——每一个操作（读文件、写网络、调用 AI 模型）都需要显式获得"能力令牌"。这种设计让 AI Agent 可以在一个严格受限的沙箱中运行，同时仍然拥有完成工作所需的全部工具。

## 1.2 设计理念

### 安全优先

Aegis 最核心的设计原则是"如果没有明确授予，就是被禁止的"。这意味着：

- 默认情况下，程序不能执行任何 I/O 操作
- 每个函数只能使用它被明确授予的能力
- 所有能力调用都会被记录到审计日志中

这与传统语言的安全模型形成鲜明对比。在 Python 中，一个 `import os; os.system("rm -rf /")` 就可以造成灾难性破坏。在 Aegis 中，这样的操作需要多个层级的能力授权，并且每一步都会被记录。

### 简洁且专注

Aegis 不是要取代 Python 或 Rust 这样的通用语言。它的目标是成为 AI Agent 编程的最佳工具。因此，语言设计刻意保持精简：

- 语法类似 Python，使用缩进表示代码块
- 只有必要的语言特性：变量、函数、控制流、能力系统
- 没有复杂的面向对象、泛型或宏系统
- 内置包涵盖 Agent 最需要的功能：文件、OS、AI 调用

### 可审计性

每一个能力调用都会被记录到审计日志中。这使得开发者可以精确了解 AI Agent 在执行过程中做了什么：

- 什么时候调用了 `out`
- 什么时候读取了文件
- 消耗了多少计算资源（燃料）
- 调用了哪个 AI 模型

这种透明度对于 AI Agent 的调试和安全审查至关重要。

## 1.3 Aegis 与其他语言的对比

| 特性 | Aegis | Python | JavaScript | Rust |
|------|-------|--------|------------|------|
| 能力安全模型 | ✓ 原生支持 | ✗ 无 | ✗ 无 | ✓ 部分 |
| 审计日志 | ✓ 内置 | ✗ 需手动 | ✗ 需手动 | ✗ 需手动 |
| 燃料限制 | ✓ 内置 | ✗ 无 | ✗ 无 | ✗ 无 |
| AI 原生支持 | ✓ 内置包 | ✗ 需第三方库 | ✗ 需第三方库 | ✗ 需第三方库 |
| 学习曲线 | 低 | 低 | 低 | 高 |
| 缩进语法 | ✓ 是 | ✓ 是 | ✗ 否 | ✗ 否 |
| 编译执行 | 字节码 VM | 解释执行 | JIT 编译 | 原生编译 |

## 1.4 适用场景

Aegis 最适合以下场景：

1. **AI Agent 沙箱**：让 LLM 生成的代码在安全受限的环境中运行
2. **自动化脚本**：需要精细控制权限的自动化任务
3. **教育用途**：教授能力安全编程范式
4. **原型开发**：快速验证 AI 驱动的应用想法

## 1.5 语言概览

下面是一个完整的 Aegis 程序，它展示了语言的核心特性：

```
fn main
  out("你好，Aegis！")
  let 名字 = in("请输入你的名字：")
  out("欢迎你，{名字}！")
```

如果你用过 Python，你会对 Aegis 的语法感到亲切。但注意这里有几个关键区别：

- `fn` 而不是 `def` 来定义函数
- `out` 而不是 `print` 来输出
- `in` 而不是 `input` 来读取输入
- 字符串插值使用 `{表达式}` 而不是 f-string 的 `{表达式}`

这些设计选择都是为了在保持熟悉感的同时，让语言更简洁、更专注。

教程包含大量可运行的代码示例。建议你在阅读的同时，打开 Aegis 的 REPL 环境实际运行这些例子。动手实践是学习编程语言最有效的方式。

- **第7章**：Aegis 最独特的能力安全系统
- **第8-9章**：内置包和 AI 功能
- **第10章**：综合示例，将所学知识融会贯通

# 第2章 快速开始

## 2.1 安装 Aegis

### 从源码编译

Aegis 使用 C++17 编写，编译需要 g++ 和 Make 工具。

```bash
# 克隆仓库
git clone <仓库地址>
cd aegis

# 编译
make

# 验证安装
./aegis --help
```

编译成功后，你会得到两个可执行文件：

- `aegis`：命令行解释器，直接运行 `.ae` 文件
- `aegis-studio`：REPL 环境，交互式编写和运行代码

### 系统要求

- 操作系统：Linux（推荐）、macOS
- 编译器：支持 C++17 的 g++（7.0+）或 clang
- 构建工具：GNU Make
- 可选：curl（用于 AI 包的真实 API 调用）

## 2.2 第一个程序

创建一个文件 `hello.ae`：

```
fn main
  out("你好，世界！")
```

运行它：

```bash
./aegis hello.ae
```

输出：

```
你好，世界！
```

恭喜！你已经运行了第一个 Aegis 程序。

### 程序结构解析

每个 Aegis 程序都需要一个 `main` 函数，它是程序的入口点。程序从 `main` 函数的第一条语句开始执行，直到函数结束。

让我们逐行分析这个最简单的程序：

- `fn main`：定义一个名为 `main` 的函数。`fn` 是函数定义的关键字。
- `  out("你好，世界！")`：调用 `out` 函数，在屏幕上输出文本。注意开头的缩进（两个空格），它表示这行代码属于 `main` 函数体。

### 关于缩进

Aegis 使用缩进来表示代码块的层级关系，这一点与 Python 类似。默认使用两个空格作为一个缩进层级。在整个教程中，我们都使用两个空格缩进。

```bash
# 错误的缩进会导致解析错误
fn main
out("缺少缩进")  # 错误！
```

```
# 正确的缩进
fn main
  out("正确缩进")
```

## 2.3 REPL 环境

Aegis 提供了一个方便的 REPL（Read-Eval-Print Loop）环境，让你可以交互式地编写和运行代码。

```bash
./aegis-studio
```

启动后，你会看到提示符 `>`，可以逐行输入代码：

```
输入 Aegis 代码，Ctrl+D 运行：
> fn main
.   out("REPL 测试")
. 
```

输入完成后，按 `Ctrl+D` 提交代码，Aegis 会立即编译并运行。

REPL 支持的编辑功能：

- **左右箭头**：在行内移动光标
- **Home/End**：跳转到行首/行尾
- **Backspace**：删除前一个字符
- **Enter**：换行（多行输入）
- **Ctrl+D**：提交并运行
- **Ctrl+C**：取消当前输入

你也可以直接运行文件：

```bash
./aegis-studio hello.ae
```

这种方式与 `./aegis hello.ae` 效果相同。

### 管道模式

REPL 也支持标准输入管道：

```bash
echo 'fn main
  out("管道测试")' | ./aegis-studio
```

## 2.4 使用 out 输出

`out` 函数是 Aegis 中最常用的输出函数。它可以接受一个参数并打印到控制台。

```
fn main
  out("Hello")
  out("World")
```

输出：

```
Hello
World
```

每次调用 `out` 都会自动换行。

### 输出多个值

你可以使用字符串插值来组合输出多个值：

```
fn main
  let 名字 = "小明"
  let 年龄 = 25
  out("我叫{名字}，今年{年龄}岁。")
```

输出：

```
我叫小明，今年25岁。
```

字符串中的 `{表达式}` 会被替换为表达式的值。

## 2.5 使用 in 输入

`in` 函数从标准输入读取一行文本。它可以接受一个可选的提示字符串参数：

```
fn main
  let 名字 = in("请输入你的名字：")
  out("你好，{名字}！")
```

运行这段程序时，Aegis 会先打印提示信息，然后等待你输入：

```
请输入你的名字：张三
你好，张三！
```

`in` 始终返回字符串类型。如果你需要数字，需要使用后续章节介绍的转换方法。

## 2.6 注释

Aegis 支持单行注释，使用 `//` 开头：

```
fn main
  // 这是一个注释
  out("Hello")  // 这也是注释
```

注释在编译时会被完全忽略，仅用于为代码提供说明。

## 2.7 第一个完整的交互程序

让我们把学到的知识结合起来，写一个完整的交互程序：

```
fn main
  out("===== 个人名片生成器 =====")
  let 姓名 = in("请输入姓名：")
  let 年龄 = in("请输入年龄：")
  let 城市 = in("请输入城市：")
  out("")
  out("===== 你的名片 =====")
  out("姓名：{姓名}")
  out("年龄：{年龄}")
  out("城市：{城市}")
  out("========================")
```

这个程序展示了：

1. `out` 输出文本
2. `in` 读取用户输入
3. 变量存储和字符串插值
4. 程序流程控制（顺序执行）

## 2.8 常见错误

### 忘记 main 函数

```
// 错误：没有 main 函数
fn foo
  out("测试")
```

错误信息：

```
缺少入口函数 main
```

每个 Aegis 程序都必须有一个 `main` 函数。

### 缩进不一致

```
fn main
  out("正确缩进")
    out("错误缩进")  // 多了一个缩进
```

错误信息：

```
缩进不一致
```

### 未闭合的字符串

```
fn main
  out("Hello)  // 缺少闭合引号
```

错误信息：

```
未闭合的字符串字面量
```

在本章中，你学会了：

- 如何使用 `out` 输出和 `in` 输入

- 如何使用 REPL 环境

# 第3章 基础语法

## 3.1 标识符

标识符是变量、函数等程序实体的名称。Aegis 的标识符规则如下：

- 可以包含字母（包括中文）、数字和下划线
- 不能以数字开头
- 区分大小写
- 不能与关键字重名

合法的标识符：

```
姓名
年龄
count
myVar
_data
版本号_2
```

非法的标识符：

```
2ndPlace    // 数字开头
my-var      // 包含连字符
let         // 关键字
```

Aegis 完全支持 Unicode 标识符，这意味着你可以使用中文、日文、韩文等语言的字符作为变量名。这对于非英语母语的开发者非常友好。

## 3.2 关键字

关键字是语言预留的、具有特殊含义的单词。它们不能用作标识符。

Aegis 的关键字列表：

| 关键字 | 用途 |
|--------|------|
| `fn` | 定义函数 |
| `let` | 声明变量 |
| `mut` | 声明可变变量 |
| `if` | 条件判断 |
| `else` | 条件分支 |
| `while` | 循环 |
| `return` | 函数返回值 |
| `true` | 布尔真值 |
| `false` | 布尔假值 |
| `and` | 逻辑与 |
| `or` | 逻辑或 |
| `not` | 逻辑非 |
| `in` | 输入函数（也是关键字） |
| `out` | 输出函数（也是关键字） |
| `package` | 导入包 |
| `requires` | 声明所需能力 |
| `break` | 跳出循环 |
| `continue` | 继续下一轮循环 |
| `int` | 整数类型注解 |
| `float` | 浮点类型注解 |
| `bool` | 布尔类型注解 |
| `str` | 字符串类型注解 |

## 3.3 语句与表达式

Aegis 区分"语句"和"表达式"：

- **表达式**：产生一个值。例如 `1 + 2`、`"hello"`、`sqrt(16)`
- **语句**：执行一个操作。例如 `let x = 5`、`out("hi")`、`return x`

大多数编程语言都有这种区分。一个有用的记忆方法：表达式可以放在等号右边，语句不行。

```
let x = (1 + 2) * 3    // ✓ 表达式可以
// let x = (let y = 5)  // ✗ 语句不行
```

## 3.4 字面量

字面量是直接在代码中写出的值。

### 整数字面量

```
42
-17
0
1000000
```

### 浮点数字面量

```
3.14
-0.5
100.0
```

### 布尔字面量

```
true
false
```

### 字符串字面量

```
"Hello, World!"
""
"中文支持"
```

字符串使用双引号包裹。Aegis 不支持单引号字符串。

### 字符串中的特殊字符

可以使用反斜杠转义：

```
"包含\"引号\"的字符串"
"第一行\n第二行"
"制表符\t在这里"
```

## 3.5 字符串插值

字符串插值是 Aegis 的一个重要特性。你可以在字符串中使用 `{表达式}` 来嵌入动态值：

```
fn main
  let 产品 = "Aegis"
  let 版本 = 1.0
  out("{产品} 语言版本 {版本}")
```

输出：

```
Aegis 语言版本 1.0
```

插值中的表达式可以是任何有效的 Aegis 表达式：

```
fn main
  let a = 10
  let b = 20
  out("a + b = {a + b}")
  out("较大值是 {if a > b {a} else {b}}")
```

注意：在插值内部可以嵌套字符串，但要小心引号的使用。

## 3.6 运算符

### 算术运算符

| 运算符 | 说明 | 示例 | 结果 |
|--------|------|------|------|
| `+` | 加法 | `5 + 3` | `8` |
| `-` | 减法 | `5 - 3` | `2` |
| `*` | 乘法 | `5 * 3` | `15` |
| `/` | 除法 | `5 / 3` | `1.666...` |
| `%` | 取模 | `5 % 3` | `2` |

### 比较运算符

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `==` | 等于 | `a == b` |
| `!=` | 不等于 | `a != b` |
| `<` | 小于 | `a < b` |
| `>` | 大于 | `a > b` |
| `<=` | 小于等于 | `a <= b` |
| `>=` | 大于等于 | `a >= b` |

比较运算符的返回值是布尔类型（`true` 或 `false`）。

### 逻辑运算符

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `and` 或 `&&` | 逻辑与 | `a > 0 and a < 10` |
| `or` 或 `||` | 逻辑或 | `a == 0 or b == 0` |
| `not` | 逻辑非 | `not a` |

### 运算符优先级

从高到低：

1. `not`（一元非）
2. `*` `/` `%`
3. `+` `-`
4. `<` `>` `<=` `>=`
5. `==` `!=`
6. `and` / `&&`
7. `or` / `||`

使用括号 `()` 可以改变优先级：

```
fn main
  out(2 + 3 * 4)      // 14
  out((2 + 3) * 4)    // 20
  out((a > 0) and (a < 10))
```

## 3.7 表达式语句

在 Aegis 中，任何表达式都可以作为语句使用。这种情况下，表达式的值会被计算但丢弃。

```
fn main
  42            // 无副作用的表达式语句（没有实际作用）
  1 + 2         // 同上
  out("有用")   // 输出函数调用，有副作用
```

通常，只有有副作用的表达式（如函数调用）才会作为表达式语句使用。

## 3.8 package 声明

`package` 声明用于导入包。它必须出现在文件的顶层（不能在函数内部）。

```
package file
package os
package ai

fn main
  // 现在可以使用这些包中的函数了
```

关于每个包的具体功能，我们将在第8章详细介绍。

## 3.9 程序结构

一个完整的 Aegis 程序由以下部分组成：

```
// 1. package 声明（可选，在文件顶部）
package file
package ai

// 2. 函数定义
fn helper
  return 42

// 3. main 函数（必需）
fn main
  out("程序开始")
  let 结果 = helper()
  out("结果：{结果}")
```

注意：

- `package` 声明必须在所有函数定义之前
- 辅助函数可以在 `main` 之前或之后定义
- 所有函数都是平级的（不支持嵌套函数）

# 第4章 变量、类型与运算符

## 4.1 变量声明

在 Aegis 中，使用 `let` 关键字声明变量：

```
let 变量名 = 值
```

### 基本用法

```
fn main
  let 姓名 = "张三"
  let 年龄 = 28
  let 身高 = 175.5
  let 是否学生 = false
  out("姓名：{姓名}")
  out("年龄：{年龄}")
  out("身高：{身高}")
  out("是否学生：{是否学生}")
```

### 类型注解

Aegis 支持可选的类型注解，使用冒号语法：

```
let 类型:变量名 = 值
```

```
fn main
  let str:姓名 = "张三"
  let int:年龄 = 28
  let float:身高 = 175.5
  let bool:是否学生 = false
```

类型注解在当前的 Aegis 版本中主要用于文档目的，编译器并不强制执行类型检查。但它们可以提高代码的可读性，并为你未来的代码审查提供帮助。

### 可变变量

默认情况下，变量是不可变的（值不能改变）。使用 `mut` 关键字声明可变变量：

```
let mut 变量名 = 初始值
```

```
fn main
  let mut 计数器 = 0
  out("初始值：{计数器}")
  计数器 = 1        // 修改值
  out("修改后：{计数器}")
  计数器 = 计数器 + 1
  out("递增后：{计数器}")
```

尝试修改不可变变量会导致编译错误：

```
fn main
  let 常量 = 10
  常量 = 20           // 错误！常量是不可变的
```

### 常见错误

```
let x = 5
x = x + 1       // 错误！x 不可变，需要 let mut
```

正确写法：

```
let mut x = 5
x = x + 1       // ✓ 正确
```

## 4.2 变量命名规范

虽然 Aegis 允许使用中文变量名，但建议遵循以下规范：

- 使用有意义的名称，让人一看就明白变量的用途
- 对于英文变量名，使用驼峰命名法（`myVariable`）或下划线命名法（`my_variable`）
- 常量（不可变变量）可以使用全大写加下划线（虽然 Aegis 没有强制约定）

好的命名：

```
let 用户名 = "张三"          // 中文命名，清晰明了
let userAge = 28              // 驼峰命名
let is_active = true          // 下划线命名
let MAX_COUNT = 100           // 常量风格
```

差的命名：

```
let a = "张三"                // 无意义的名称
let 变量 = 28                 // 过于泛泛
let xyz123 = true             // 难以理解的名称
```

## 4.3 基本数据类型

Aegis 支持四种基本数据类型：

### 整数 (int)

整型数值，可以是正数、负数或零：

```
let a = 42
let b = -17
let c = 0
let d = 1000000
```

整数的运算：

```
let x = 10
let y = 3
out(x + y)    // 13
out(x - y)    // 7
out(x * y)    // 30
out(x / y)    // 3.333...  （自动转为浮点）
out(x % y)    // 1
```

注意：整数除法会返回浮点数结果。

### 浮点数 (float)

带小数点的数值：

```
let pi = 3.14159
let e = 2.71828
let 温度 = -5.5
let 零 = 0.0
```

浮点数的运算：

```
let a = 5.5
let b = 2.0
out(a + b)    // 7.5
out(a - b)    // 3.5
out(a * b)    // 11.0
out(a / b)    // 2.75
```

### 布尔值 (bool)

表示真或假的值，只有两个可能取值：

```
let 是 = true
let 否 = false
```

布尔值通常用于条件判断：

```
let 温度 = 30
let 太热了 = 温度 > 35
if 太热了
  out("开空调")
else
  out("温度适宜")
```

### 字符串 (str)

文本数据，使用双引号包裹：

```
let 问候 = "你好"
let 空字符串 = ""
let 带引号 = "他说：\"你好\""
let 多行 = "第一行\n第二行"
```

字符串可以包含任意 Unicode 字符：

```
let 中文 = "你好世界"
let 日文 = "こんにちは"
let 表情 = "😊👍"
```

### 字符串操作

字符串连接通过插值实现：

```
let 名 = "张"
let 姓 = "三"
let 全名 = "{名}{姓}"    // "张三"
```

字符串比较使用 `==` 和 `!=`：

```
let a = "hello"
let b = "hello"
let c = "world"
out(a == b)    // true
out(a == c)    // false
out(a != c)    // true
```

字符串可以使用包函数进行操作（见第8章）。

## 4.4 类型转换

### 字符串转整数

使用 `to_int()` 函数（来自 `str` 包，但请注意当前版本 `str` 包已移除，需要自定义转换或使用内置转换）：

实际上，在当前的 Aegis 版本中，没有内置的类型转换函数。数值运算会在必要时自动进行类型提升（如整数除法产生浮点数）。

### 数值运算中的自动类型提升

```
let a = 10        // int
let b = 3.5      // float
out(a + b)       // 13.5 (int 自动提升为 float)
out(a * b)       // 35.0
```

当整数和浮点数混合运算时，整数会自动提升为浮点数。

## 4.5 常量表达式

由于变量默认不可变，你可以将常量定义为：

```
let 税率 = 0.15
let 最大重试次数 = 3
let 应用名称 = "Aegis"
```

这些"常量"在声明后不能被修改，起到了常量的作用。

## 4.6 作用域

变量的作用域由它所在的代码块决定。代码块由缩进定义：

```
fn main
  let x = 10          // x 在 main 函数作用域内
  if x > 5
    let y = 20        // y 只在 if 块内有效
    out(y)            // ✓ 可以访问
  out(x)              // ✓ 可以访问
  // out(y)           // 错误！y 不在此作用域
```

在内部作用域中，不能访问外部作用域的变量（当前实现限制）。

```
fn main
  let x = 10
  if true
    let y = x + 5     // ✓ 可以读取外部变量
    out(y)            // 15
```

## 4.7 变量的阴影

在同一个作用域中不能重复声明同名变量：

```
fn main
  let x = 10
  // let x = 20      // 错误！重复声明
```

## 4.8 运算符的详细用法

### 算术运算符

```
fn main
  out(10 + 3)       // 13
  out(10 - 3)       // 7
  out(10 * 3)       // 30
  out(10 / 3)       // 3.33333
  out(10 % 3)       // 1

  // 浮点数运算
  out(10.0 / 3.0)   // 3.33333
  out(10.5 % 3.2)   // 0.9（浮点取模）
```

### 比较运算符

```
fn main
  out(5 == 5)       // true
  out(5 != 5)       // false
  out(5 < 10)       // true
  out(5 > 10)       // false
  out(5 <= 5)       // true
  out(5 >= 10)      // false

  // 字符串比较
  out("a" == "a")   // true
  out("a" != "b")   // true
```

### 逻辑运算符

```
fn main
  let 年龄 = 25
  let 有驾照 = true

  if 年龄 >= 18 and 有驾照
    out("可以开车")

  if 年龄 < 18 or not 有驾照
    out("不能开车")

  let 是晴天 = true
  if not 是晴天
    out("带伞")
```

运算符 `and` 和 `or` 使用短路求值：

```
fn main
  let x = 0
  // 如果 x == 0，右侧不会执行（不会有除以零错误）

    out("不会执行到这里")
```

```
fn main

  let 折扣率 = 0.2
  let 会员折扣 = 0.1
  let 税率 = 0.08

  let 折扣后价格 = 原始价格 * (1 - 折扣率)
  let 会员价 = 折扣后价格 * (1 - 会员折扣)

  out("折扣后：{折扣后价格}元")
  out("会员价：{会员价}元")
  out("含税最终价：{最终价格}元")

  let 是否免运费 = 最终价格 > 100
  if 是否免运费
    out("免运费")
  else
    out("运费：10元")
```

## 4.10 小结

本章详细介绍了 Aegis 的变量和类型系统：

- 使用 `let` 声明变量，`let mut` 声明可变变量
- 四种基本数据类型：int、float、bool、str
- 可选的类型注解语法
- 运算符的详细用法和优先级
- 变量的作用域规则
- 类型在运算中的自动提升

# 第5章 控制流

## 5.1 条件语句：if/else

条件语句让程序可以根据不同的条件执行不同的代码。

### 基本 if

```
fn main
  let 分数 = 85
  if 分数 >= 60
    out("及格")
```

### if/else

```
fn main
  let 分数 = 45
  if 分数 >= 60
    out("及格")
  else
    out("不及格")
```

### 多重条件

Aegis 不支持 `else if` 语法，你可以使用嵌套的 if 来实现多重条件：

```
fn main
  let 分数 = 85
  if 分数 >= 90
    out("优秀")
  else
    if 分数 >= 80
      out("良好")
    else
      if 分数 >= 70
        out("中等")
      else
        if 分数 >= 60
          out("及格")
        else
          out("不及格")
```

或者使用逻辑运算符组合条件：

```
fn main
  let 分数 = 85
  if 分数 >= 90
    out("优秀")
  if 分数 >= 80 and 分数 < 90
    out("良好")
  if 分数 >= 70 and 分数 < 80
    out("中等")
  if 分数 >= 60 and 分数 < 70
    out("及格")
  if 分数 < 60
    out("不及格")
```

### 条件表达式

在 Aegis 中，`if/else` 也可以作为表达式使用，返回值：

```
fn main
  let x = 10
  let 结果 = if x > 5 { "大" } else { "小" }
  out(结果)    // "大"
```

注意：当在字符串插值中使用条件表达式时，需要使用 `{}` 包裹：

```
fn main
  let x = 10
  out("x {if x > 5 {">5"} else {"<=5"}}")
```

## 5.2 while 循环

`while` 循环在条件为真时重复执行代码块。

### 基本 while

```
fn main
  let mut 计数 = 0
  while 计数 < 5
    out("计数：{计数}")
    计数 = 计数 + 1
```

输出：

```
计数：0
计数：1
计数：2
计数：3
计数：4
```

### 无限循环

虽然 Aegis 没有 `loop` 关键字，但你可以使用 `while true` 创建无限循环：

```
fn main
  let mut 计数 = 0
  while true
    out("循环第{计数}次")
    计数 = 计数 + 1
    if 计数 >= 5
      break
```

## 5.3 break 和 continue

### break

`break` 立即退出当前循环：

```
fn main
  let mut i = 0
  while i < 100
    if i * i > 50
      out("第一个平方大于50的数是：{i}")
      break
    i = i + 1
```

### continue

`continue` 跳过当前迭代的剩余部分，进入下一次循环：

```
fn main
  let mut i = 0
  while i < 10
    i = i + 1
    if i % 2 == 0
      continue
    out("{i} 是奇数")
```

输出：

```
1 是奇数
3 是奇数
5 是奇数
7 是奇数
9 是奇数
```

## 5.4 嵌套控制流

控制流可以任意嵌套组合：

```
fn main
  let mut i = 0
  while i < 3
    let mut j = 0
    while j < 3
      let 乘积 = i * j
      if 乘积 > 0
        out("{i} * {j} = {乘积}")
      j = j + 1
    i = i + 1
```

## 5.5 实用示例：猜数字游戏

```
fn main
  out("===== 猜数字游戏 =====")
  let 目标 = 42
  let mut 猜测次数 = 0

  while true
    let 输入 = in("请输入你猜的数字：")
    let mut 猜测 = 0

    // 简单解析输入（这里简化处理）
    if 输入 == "42"
      猜测 = 42
    else
      if 输入 == "0"
        猜测 = 0
      else
        out("请输入有效数字")
        continue

    猜测次数 = 猜测次数 + 1

    if 猜测 == 目标
      out("恭喜！你猜对了！共猜了{猜测次数}次")
      break
    else
      if 猜测 < 目标
        out("猜小了")
      else
        out("猜大了")
```

## 5.6 实用示例：计算器

```
fn main
  out("===== 简单计算器 =====")
  out("支持的运算：+ - * /")
  while true
    let 输入 = in("输入表达式（如 2 + 3），输入 q 退出：")
    if 输入 == "q"
      out("再见！")
      break

    // 这里简化解析，实际应有更完善的解析
    // 我们直接演示几个固定计算
    let a = 10
    let b = 5
    out("{a} + {b} = {a + b}")
    out("{a} - {b} = {a - b}")
    out("{a} * {b} = {a * b}")
    out("{a} / {b} = {a / b}")
```

## 5.7 常见模式

### 输入验证

```
fn main
  let mut 有效输入 = false
  while not 有效输入
    let 输入 = in("请输入 y 或 n：")
    if 输入 == "y" or 输入 == "n"
      有效输入 = true
      out("你输入了：{入力}")
    else
      out("无效输入，请重试")
```

### 菜单循环

```
fn main
  while true
    out("")
    out("===== 菜单 =====")
    out("1. 查看信息")
    out("2. 修改设置")
    out("3. 退出")
    let 选择 = in("请选择：")
    if 选择 == "1"
      out("你选择了查看信息")
    else
      if 选择 == "2"
        out("你选择了修改设置")
      else
        if 选择 == "3"
          out("再见！")
          break
        else
          out("无效选择")
```

## 5.8 控制流中的注意事项

### 空块

Aegis 不允许空的代码块。如果你需要一个什么都不做的条件分支，可以使用占位语句：

```
fn main
  let x = 10
  if x > 5
    // 这里不能完全空着
    let _ = 0   // 占位
```

### 避免死循环

确保你的循环条件最终会变为假，否则程序会一直运行直到燃料耗尽：

```
fn main
  let mut x = 0
  while x < 10
    // 忘记递增 x 会导致死循环！
    // x = x + 1  // 如果注释掉这行，程序会永远运行
```

如果出现死循环，VM 会在燃料耗尽后抛出异常并终止。

## 5.9 小结

本章介绍了 Aegis 的控制流机制：

- 使用 `if/else` 进行条件判断
- 使用 `while` 实现循环
- 使用 `break` 和 `continue` 控制循环流程
- 嵌套控制流的使用
- 实用的控制流模式

# 第6章 函数

## 6.1 函数定义

函数是 Aegis 中组织和复用代码的基本单位。

### 基本语法

```
fn 函数名(参数列表)
  函数体
```

### 简单函数

```
fn 打招呼
  out("你好！")

fn main
  打招呼()    // 调用函数
```

### 带参数的函数

```
fn 问候(姓名)
  out("你好，{姓名}！")

fn main
  问候("张三")
  问候("李四")
```

参数可以有类型注解：

```
fn 加法(int:a, int:b)
  out("{a} + {b} = {a + b}")

fn main
  加法(3, 5)
```

### 参数的默认值

当前版本不支持默认参数值。如果你需要类似的功能，可以在函数内部处理：

```
fn 问候(姓名)
  if 姓名 == ""
    姓名 = "朋友"
  out("你好，{姓名}！")

fn main
  问候("")     // 输出：你好，朋友！
  问候("张三")  // 输出：你好，张三！
```

## 6.2 返回值

使用 `return` 关键字从函数返回一个值。

### 基本返回值

```
fn 加法(a, b)
  return a + b

fn main
  let 结果 = 加法(3, 5)
  out("3 + 5 = {结果}")
```

### 返回值类型注解

```
fn 加法(a: int, b: int) -> int
  return a + b
```

`-> int` 表示这个函数返回一个整数。类型注解在当前的 Aegis 版本中主要用于文档目的。

### 提前返回

```
fn 检查年龄(年龄)
  if 年龄 < 0
    return "无效年龄"
  if 年龄 < 18
    return "未成年人"
  return "成年人"

fn main
  out(检查年龄(-5))
  out(检查年龄(15))
  out(检查年龄(25))
```

### 无返回值

如果没有 `return` 语句，函数隐式返回 `nil`：

```
fn 打招呼(姓名)
  out("你好，{姓名}！")
  // 隐式返回 nil

fn main
  let 结果 = 打招呼("张三")
  out(结果)   // nil
```

## 6.3 函数调用

### 基本调用

```
函数名(参数1, 参数2, ...)
```

### 嵌套调用

```
fn 平方(x)
  return x * x

fn 两倍(x)
  return x * 2

fn main
  let 结果 = 两倍(平方(3))
  out("结果：{結果}")    // 18
```

### 在表达式中调用

```
fn main
  let x = 5
  out("5 的平方是 {平方(5)}")
  out("比 5 大 3 的数的平方是 {平方(x + 3)}")
```

## 6.4 函数的完整示例

### 温度转换

```
fn 摄氏转华氏(c)
  return c * 9 / 5 + 32

fn 华氏转摄氏(f)
  return (f - 32) * 5 / 9

fn main
  out("===== 温度转换器 =====")
  let 摄氏 = 25
  let 华氏 = 摄氏转华氏(摄氏)
  out("{摄氏}°C = {华氏}°F")

  let 输入华氏 = 77
  let 输出摄氏 = 华氏转摄氏(输入华氏)
  out("{输入华氏}°F = {输出摄氏}°C")
```

### 斐波那契数列

```
fn 斐波那契(n)
  if n <= 1
    return n
  let mut a = 0
  let mut b = 1
  let mut i = 2
  while i <= n
    let temp = a + b
    a = b
    b = temp
    i = i + 1
  return b

fn main
  let mut i = 0
  while i <= 10
    out("F({i}) = {斐波那契(i)}")
    i = i + 1
```

输出：

```
F(1) = 1
F(2) = 1
F(3) = 2
F(4) = 3
F(5) = 5
F(6) = 8
F(7) = 13
F(8) = 21
F(9) = 34
F(10) = 55
```

### 素数判断

```
fn 是素数(n)

    return false
  let mut i = 2
  while i * i <= n
    if n % i == 0
      return false
    i = i + 1
  return true

fn main

  while i <= 30
    if 是素数(i)
      out("{i} 是素数")

```

```
fn 计算(a, b, 操作)

```

## 6.7 使用函数构建模块化程序

将程序分解为小的、可复用的函数是良好编程实践的核心。

### 示例：学生成绩管理系统

```
fn 计算等级(分数)
  if 分数 >= 90
    return "优秀"
  if 分数 >= 80
    return "良好"
  if 分数 >= 70
    return "中等"
  if 分数 >= 60
    return "及格"
  return "不及格"

fn 计算平均分(总分, 科目数)
  return 总分 / 科目数

fn 显示成绩单(姓名, 语文, 数学, 英语)
  let 总分 = 语文 + 数学 + 英语
  let 平均分 = 计算平均分(总分, 3)

  out("===== {姓名} 的成绩单 =====")
  out("语文：{语文} ({计算等级(语文)})")
  out("数学：{数学} ({计算等级(数学)})")
  out("英语：{英语} ({计算等级(英语)})")
  out("总分：{总分}")
  out("平均分：{平均分}")
  out("总评：{计算等级(平均分)}")

fn main
  显示成绩单("张三", 85, 92, 78)
  out("")
  显示成绩单("李四", 95, 88, 91)
```

## 6.8 递归

Aegis 支持递归函数调用：

```
fn 阶乘(n)
  if n <= 1
    return 1
  return n * 阶乘(n - 1)

fn main
  out("5! = {阶乘(5)}")    // 120
  out("10! = {阶乘(10)}")  // 3628800
```

注意：递归深度受燃料限制。过深的递归可能导致燃料耗尽。

### 递归的替代方案

对于可以迭代实现的问题，优先使用循环而不是递归：

```
// 使用循环实现的阶乘（更高效）
fn 阶乘_iter(n)
  let mut 结果 = 1
  let mut i = 2
  while i <= n
    结果 = 结果 * i
    i = i + 1
  return 结果
```

## 6.9 函数设计最佳实践

1. **单一职责原则**：一个函数只做一件事
2. **有意义的名称**：函数名应该说明它做什么
3. **限制参数数量**：参数太多（超过3-4个）说明函数可能太复杂
4. **避免过长的函数**：如果一个函数超过30行，考虑拆分成多个小函数

```
// 好的设计
fn 验证年龄(年龄)
  return 年龄 >= 0 and 年龄 <= 150

fn 计算折扣(价格, 会员等级)
  if not 验证年龄(价格)
    return 0
  if 会员等级 == "金卡"
    return 价格 * 0.8
  if 会员等级 == "银卡"
    return 价格 * 0.9
  return 价格

// 不好的设计（太长、职责不明确）
fn 处理订单(客户, 商品, 数量, 价格, 地址, 支付方式)
  // 几十行代码混杂了验证、计算、输出等不同职责
```

# 第7章 能力系统与安全模型

## 7.1 什么是能力安全

能力安全（Capability Security）是一种安全模型，其核心思想是：**程序没有默认权限，每个权限都必须被显式授予**。

在传统安全模型中，程序要么拥有全部权限（如普通桌面应用），要么通过"用户身份"来获得权限（如 Linux 的 UID 系统）。这些模型在面对 AI Agent 时存在根本性问题：

- AI Agent 可能被提示注入攻击
- 无法精确控制 Agent 的每个操作
- 缺乏细粒度的审计

能力安全模型解决了这些问题。每个"能力"都是一个不可伪造的令牌。程序必须持有令牌才能执行对应操作。

## 7.2 Aegis 的能力模型

Aegis 的能力模型基于以下概念：

### 能力（Capability）

能力是一个代表特定权限的抽象概念。在 Aegis 中，`io` 是一个能力，它允许函数执行输入/输出操作。

### 能力声明

函数通过 `requires` 关键字声明它需要的能力：

```
fn 安全操作() requires io: IO
  out("这个函数有 IO 权限")
```

但目前，在 Aegis 的简化实现中，`out` 和 `in` 是全局可用的，不需要显式声明。

### 能力的传播

当一个函数调用另一个函数时，能力会自动传播：

```
fn 内部函数() 
  out("内部函数可以调用 out")

fn 外部函数()
  内部函数()    // io 能力自动传播到内部函数
```

### 审计日志

每次能力调用都会被记录到审计日志中：

```
[audit] io.print called
[audit] io.read_line called
```

## 7.3 包与能力

当 Aegis 的程序变得越来越复杂时，你可能需要访问更强大的能力。内置包提供了这种能力。

### package 声明

使用 `package` 关键字导入功能包：

```
package file     // 文件操作能力
package os       // 操作系统能力
package ai       // AI 调用能力
```

每个包提供一组相关的函数。

### 包函数的安全含义

当前版本的 Aegis 中，包函数默认可用（不要求显式能力声明）。但在设计上，每个包未来都可以被细粒度地控制。

## 7.4 燃料系统

燃料（Fuel）是 Aegis 的另一个安全机制。它限制了程序可以执行的计算量：

### 工作原理

- 每个虚拟机指令消耗一定量的燃料
- 程序在燃料耗尽时被强制终止
- 默认燃料上限为 1,000,000 单位

### 为什么需要燃料

```
// 如果程序进入无限循环，燃料会耗尽
fn main
  let mut x = 0
  while true
    x = x + 1    // 每次循环消耗燃料
```

如果没有燃料限制，这样的程序会永远运行下去，消耗系统资源。燃料机制确保了：

- 即使是恶意程序也不能无限运行
- AI Agent 的每次执行都有确定的资源上限
- 宿主系统可以安全地运行不可信代码

## 7.5 实际的安全场景

### 场景1：运行不可信的 AI 生成代码

想象你的 AI Agent 生成了以下代码：

```
fn main
  // AI 生成的文件删除操作
  out("开始清理")
  write("/etc/passwd", "恶意内容")
```

如果代码在标准 Python 中运行，这会造成严重破坏。但在 Aegis 中：

- 如果程序没有 `package file`，`write` 函数不可用
- 即使有 `package file`，文件操作也受到操作系统权限限制
- 所有文件操作都会被审计日志记录

### 场景2：控制 AI Agent 的资源使用

```
fn main
  let mut 计数 = 0
  while 计数 < 1000000
    计数 = 计数 + 1
  out("完成")
```

这个程序会消耗大量燃料。如果燃料上限设置为 100,000，程序会在完成之前被终止，保护宿主系统不受资源耗尽攻击。

## 7.6 审计日志详解

审计日志是调试和安全审查的重要工具。

### 查看审计日志

每次运行程序时，审计信息会输出到 stderr：

```
fn main
  out("hello")
  out("world")
```

运行输出：

```
[audit] io.print called
hello
[audit] io.print called
world
```

### 审计信息的解读

审计日志记录以下信息：

- 操作类型（print、read_line 等）
- 操作时间（通过日志顺序推断）
- 操作的来源

### 审计日志的用途

1. **调试**：追踪程序的执行流程
2. **安全检查**：确认程序没有进行未授权的操作
3. **行为分析**：了解 AI Agent 的执行模式
4. **取证**：在安全事件发生后重建攻击路径

## 7.7 安全编程最佳实践

### 最小权限原则

只授予程序完成任务所需的最小能力集：

```
// 只需要输出，不需要文件操作
package agent

fn main
  out("Hello")
```

### 能力分离

将需要不同能力的操作分离到不同函数中：

```
fn 纯计算(a, b)
  return a + b

fn 需要IO()
  out("计算结果：{纯计算(10, 20)}")
```

### 输入验证

永远不要信任外部输入：

```
fn main
  let 输入 = in("请输入数字：")
  // 处理输入前验证
  if 输入 == "1" or 输入 == "2" or 輸入 == "3"
    out("有效输入：{輸入}")
  else
    out("无效输入")
```

## 7.8 安全模型对比

### Aegis vs 传统沙箱

| 特性 | Aegis | Docker | seccomp | Java Security |

## 7.9 未来规划：扩展能力系统

在未来的版本中，Aegis 的计划包括：

# 第8章 内置包详解

## 8.1 包系统概述

Aegis 通过包系统扩展语言的功能。包是一组相关函数的集合，通过 `package` 关键字导入。

### 使用包

```
package file     // 导入文件操作包
package os       // 导入操作系统包
package ai       // 导入 AI 包

fn main
  out("当前目录：{cwd()}")
  write("/tmp/test.txt", "Hello")
  let 内容 = read("/tmp/test.txt")
  out(内容)
```

### 包的命名

当前版本中，包名是简单的字符串（如 `file`、`os`、`ai`）。未来可能支持带路径的包名（如 `std.file`）。

## 8.2 file 包

`file` 包提供文件系统操作。

### file.read

读取文件全部内容：

```
let 内容 = read("路径/文件名.txt")
```

如果文件不存在，会抛出异常：

```
let 内容 = read("不存在的文件.txt")  // 运行时错误
```

### file.write

写入文件（覆盖已有内容）：

```
write("路径/文件名.txt", "要写入的内容")
```

如果文件所在的目录不存在，会抛出异常。

### file.exists

检查文件是否存在：

```
let 是否存在 = exists("路径/文件名.txt")
if 是否存在
  out("文件存在")
else
  out("文件不存在")
```

返回值是布尔类型。

### file.delete

删除文件：

```
delete("路径/文件名.txt")
```

如果文件不存在，不会抛出异常（静默失败）。

### file.append

追加内容到文件末尾：

```
append("路径/文件名.txt", "要追加的内容")
```

如果文件不存在，会创建新文件。

### 完整示例

```
package file

fn main
  // 写入文件
  write("/tmp/日记.txt", "2024年1月1日\n")
  
  // 追加内容
  append("/tmp/日记.txt", "今天学习了 Aegis 语言！\n")
  
  // 检查文件
  if exists("/tmp/日记.txt")
    out("日记文件已创建")
  
  // 读取文件
  let 内容 = read("/tmp/日记.txt")
  out("日记内容：")
  out(内容)
  
  // 删除文件
  delete("/tmp/日记.txt")
  out("日记文件已删除")
```

### 文件路径注意事项

- 使用绝对路径（如 `/tmp/file.txt`）或相对路径（如 `data/file.txt`）
- 当前工作目录由运行环境决定
- 使用 `os.cwd()` 可以获取当前工作目录

## 8.3 os 包

`os` 包提供操作系统级别的功能。

### os.env

获取环境变量值：

```
let 路径 = env("PATH")
out("PATH = {路径}")

let home = env("HOME")
out("HOME = {home}")
```

如果环境变量不存在，返回空字符串。

### os.cwd

获取当前工作目录：

```
let 当前目录 = cwd()
out("当前目录：{当前目录}")
```

### os.exit

退出程序并返回退出码：

```
if 错误
  exit(1)    // 非零退出码表示错误
else
  exit(0)    // 零表示成功
```

注意：`exit` 会立即终止整个程序。

### 完整示例

```
package os

fn main
  out("===== 系统信息 =====")
  
  // 用户信息
  out("用户：{env("USER")}")
  out("HOME：{env("HOME")}")
  
  // 工作目录

  let 模式 = env("AEGIS_MODE")

    模式 = "default"
  out("运行模式：{模式}")
  
  // 路径信息
  let path = env("PATH")
  out("PATH：{path}")
```

### 环境变量的用途

环境变量的常见用途：

1. **配置**：通过环境变量传递配置信息（如 API key、数据库地址等）

3. **调试**：通过环境变量控制调试输出


- 在写入文件前检查文件是否存在（如果需要防止覆盖）

```
fn main
  let 用户输入 = in("请输入文件名：")
  // 不要直接使用用户输入
  // write("/var/data/{用户輸入}", 内容)  // 不安全！
  
  // 应该限制在安全目录内
  let 安全路径 = "/tmp/{用户输入}"
  write(安全路径, 内容)
```

### 环境变量的安全风险

- 不要在代码中硬编码敏感信息（如 API key）
- 使用环境变量传递敏感信息
- 注意环境变量可能被子进程继承

```
// 安全：使用环境变量
let key = env("API_KEY")

// 不安全：硬编码
// let key = "sk-1234567890"
```

### AI 调用的安全风险

- AI 模型可能生成不安全的内容
- 不要直接执行 AI 生成的代码而不审查
- 对 AI 的输出进行适当的后处理

```
// 使用系统提示词限制 AI 的行为
let 回复 = chat("gpt-4o-mini",
  "你是一个安全的助手。如果用户要求你生成危险内容，礼貌地拒绝。",
  用户消息)
```

本章详细介绍了 Aegis 的三个内置包：

- **file 包**：文件读写、检查、删除和追加操作
- **os 包**：环境变量、工作目录和进程控制
- **ai 包**：AI 聊天调用（真实模式和 mock 模式）

同时讨论了包使用中的安全注意事项。

# 第9章 AI 编程与 LangChain 风格

## 9.1 LangChain 简介

LangChain 是一个流行的 AI 应用开发框架。它的核心思想是：将 AI 模型与外部工具和数据源连接起来，构建更强大的 AI 应用。

Aegis 借鉴了 LangChain 的设计理念，但以更简洁、更安全的方式实现了类似的功能。

## 9.2 基础 AI 调用

最简单的 AI 调用就是直接使用 `chat` 函数：

```
package ai

fn main
  let 答案 = chat("gpt-4o-mini",
    "你是一个助手",
    "什么是人工智能？")
  out(答案)
```

这与 LangChain 中的 `LLMChain` 类似，但更加直接。

## 9.3 模板化提示词

LangChain 中的 `PromptTemplate` 可以用 Aegis 的字符串插值实现：

```
package ai

fn main
  let 主题 = "量子计算"
  let 风格 = "通俗易懂"
  let 字数 = 200
  
  let 提示 = "请用{风格}的方式解释{主题}，控制在{字数}字以内"
  let 答案 = chat("gpt-4o-mini", "你是一个科普作家", 提示)
  out(答案)
```

## 9.4 链式调用

LangChain 中的 `Chain` 可以通过函数组合实现：

```
package ai

fn 生成大纲(主题)
  return chat("gpt-4o-mini",
    "你是一个写作助手，输出简洁的提纲",
    "为主题'{主题}'生成一个文章提纲")

fn 扩写章节(提纲, 章节)
  return chat("gpt-4o-mini",
    "你是一个作者，根据提纲扩写指定章节",
    "提纲：{提纲}\n\n请扩写章节：{章节}")

fn main
  let 主题 = "人工智能的未来"
  
  out("第一步：生成提纲")
  let 提纲 = 生成大纲(主题)
  out(提纲)
  
  out("")
  out("第二步：扩写第一章")
  let 第一章 = 扩写章节(提纲, "第一章")
  out(第一章)
```

这种通过函数组合的方式，实现了类似 LangChain 的链式处理流程。

## 9.5 AI Agent 模式

AI Agent 的核心是：AI 模型分析任务，调用工具，处理结果，继续循环直到任务完成。

### 简单 Agent 实现

```
package ai
package file

fn 搜索知识(查询)
  // 模拟知识搜索
  let 知识库 = "Aegis 是一门安全的 AI Agent 语言"
  if 查询 == "Aegis"
    return 知识库
  return "未找到相关信息"

fn main
  let 任务 = in("请输入你的任务：")
  let mut 结果 = ""
  let mut 步骤 = 0
  
  while 步骤 < 3 and 结果 == ""
    步骤 = 步骤 + 1
    out("第{步骤}步：AI 正在分析...")
    
    let 分析 = chat("gpt-4o-mini",
      "你是一个 AI Agent。分析以下任务，决定需要什么信息。"
      + "如果需要在知识库中搜索，回复：SEARCH:查询词",
      任务)
    
    out("AI 分析：{分析}")
    
    // 检查 AI 是否需要搜索
    if 分析中的字符串(分析, "SEARCH:")
      let 查询 = 提取查询(分析, "SEARCH:")
      let 知识 = 搜索知识(查询)
      out("找到信息：{知识}")
      任务 = "{任务}\n参考信息：{知识}"
    else
      结果 = 分析
  
  if 结果 != ""
    out("最终结果：{結果}")
  else
    out("无法完成：超出最大步骤数")
```

注意：上面的示例中 `分析中的字符串` 和 `提取查询` 是伪代码，实际 Aegis 中需要自定义实现。

## 9.6 工具使用

## 9.7 LangChain 风格的代理流程

一个更完整的 LangChain 风格代理流程：

```
package ai

fn 思考(任务, 上下文)
  let 提示 = "任务：{任务}\n上下文：{上下文}\n请思考下一步应该做什么。"
  + "如果需要更多信息，请回答：需要：[信息需求]"
  + "如果可以回答，请回答：答案：[最终答案]"
  
  return chat("gpt-4o-mini", "你是一个谨慎的思考者。", 提示)

fn 执行(行动)
  // 执行具体的行动（这里简化）
  if 行动 == "搜索知识"
    return "找到相关信息：Aegis 支持能力安全模型"
  return "无法执行行动：{行动}"

fn main
  let 任务 = in("请输入任务：")
  let mut 上下文 = ""
  let mut 步骤 = 0
  
  while 步骤 < 5
    步骤 = 步骤 + 1
    out("\n===== 步骤 {步骤} =====")
    
    let 思考结果 = 思考(任务, 上下文)
    out("思考：{思考结果}")
    
    // 检查是否得到最终答案
    if 思考结果中的字符串(思考结果, "答案：")
      let 答案 = 提取答案(思考结果, "答案：")
      out("\n最终答案：{答案}")
      break
    
    // 如果需要更多信息，执行行动
    if 思考结果中的字符串(思考结果, "需要：")
      let 需求 = 提取需求(思考结果, "需要：")
      out("需要信息：{需求}")
      let 信息 = 执行(需求)
      out("获取到：{信息}")
      上下文 = "{上下文}\n{需求}：{信息}"
```

注意：上面的示例中使用了伪函数 `思考结果中的字符串`、`提取答案` 和 `提取需求`。在实际 Aegis 中，你需要用字符串操作来实现这些功能。

## 9.8 与 LangChain 的对比

| 特性 | LangChain (Python) | Aegis |
|------|-------------------|-------|
| LLM 调用 | `llm.invoke()` | `chat()` |
| 提示词模板 | `PromptTemplate` | 字符串插值 |
| 链 | `LLMChain` | 函数组合 |
| Agent | `AgentExecutor` | while 循环 + 函数 |
| 工具 | `@tool` 装饰器 | 普通函数 |
| 记忆 | `ConversationBufferMemory` | 变量 + 上下文参数 |
| 安全 | 无内建 | 能力安全 + 燃料 + 审计 |

### 1. 精心设计系统提示词

系统提示词决定了 AI 的行为质量：

```
  + "包含必要的 import 语句和注释。如果代码有潜在的问题，请指出。"

```

### 2. 处理 AI 输出的不确定性

AI 的输出不是确定性的，需要适当的容错处理：

```
fn 安全调用(模型, 系统提示, 用户消息)
  let 结果 = chat(模型, 系统提示, 用户消息)
  if 结果 == "" or 结果 == "(无响应)"
    结果 = "抱歉，我暂时无法处理这个请求。"
  return 结果
```

### 3. 错误重试

网络错误或 API 限流可能导致调用失败：

```
fn 重试调用(模型, 系统提示, 用户消息, 最大重试次数)
  let mut 尝试 = 0
  while 尝试 < 最大重试次数
    let 结果 = chat(模型, 系统提示, 用户消息)
    if 结果中的字符串(结果, "error") or 结果 == "(无响应)"
      尝试 = 尝试 + 1
      out("重试第{尝试}次...")
    else
      return 结果
  return "调用失败，请稍后重试"
```

### 4. 限制 AI 调用频率

避免过于频繁的 AI 调用：

```
fn 批量处理(消息列表)
  let mut 结果列表 = ""
  let mut i = 0
  while i < 长度(消息列表)
    let 消息 = 获取消息(消息列表, i)
    let 结果 = chat("gpt-4o-mini", "处理消息", 消息)
    结果列表 = "{结果列表}\n{消息} -> {结果}"
    i = i + 1
    // 每5条消息后等待（通过空循环消耗时间）
    if i % 5 == 0
      out("处理了{i}条消息，继续...")
  return 结果列表
```

```
    out("3. 生成摘要")
    out("4. 退出")
    
    let 选择 = in("请选择：")
    
    if 选择 == "1"
      收集用户反馈()
    else
      if 选择 == "2"
        分析反馈()
      else
        if 选择 == "3"
          生成摘要()
        else
          if 选择 == "4"
            out("再见！")
            break
          else
            out("无效选择")
```

## 10.5 最佳实践总结

### 代码组织

1. **使用 `package` 声明清晰表达依赖**

```
package file    // 明确表明这个程序需要文件操作
package ai      // 明确表明这个程序需要 AI 能力
```

2. **函数放在 main 之前，便于阅读**

```
fn 辅助函数1()
  // ...

fn 辅助函数2()
  // ...

fn main
  // 主逻辑
```

3. **使用有意义的命名**

```
fn 计算折扣价格(原价, 折扣率)    // 好
fn calc(a, b)                    // 差
```

### 错误处理

4. **验证输入**

```
fn main
  let 输入 = in("请输入数字：")
  if 输入 == ""
    out("输入不能为空")
    exit(1)
```

5. **检查文件操作结果**

```
fn main
  let 路径 = in("请输入文件路径：")
  if not exists(路径)
    out("文件不存在：{路径}")
    exit(1)
  let 内容 = read(路径)
```

### 安全实践

6. **最小权限原则**

```
// 只需要输出时，只导入必要的包
fn main
  out("Hello")
```

7. **注意路径安全**

```
fn 安全写入(目录, 文件名, 内容)
  let 完整路径 = "{目录}/{文件名}"
  write(完整路径, 内容)
```

### AI 编程实践

8. **精心设计提示词**

```
fn AI调用(任务)
  let 提示 = "你是一个{角色}。{约束}。请{指令}。"
  return chat(模型, 提示, 任务)
```

9. **处理 AI 输出的不确定性**

```
fn 安全AI调用(模型, 系统提示, 用户消息)
  let 结果 = chat(模型, 系统提示, 用户消息)
  if 结果 == "" or 结果 == "(无响应)"
    结果 = "无法获取 AI 回复"
  return 结果
```

10. **控制资源使用**

```
fn main
  // 限制循环次数
  let mut i = 0
  while i < 100
    // 处理...
    i = i + 1
```

### 性能考虑

11. **避免不必要的字符串操作**

```
// 好的方式
let 结果 = "{名称}：{值}"

// 避免在循环中频繁拼接长字符串
```

12. **使用局部变量减少重复计算**

```
fn 复杂计算(值)
  let 临时 = 值 * 值    // 只计算一次
  return 临时 + 临时 / 2
```

## 10.6 常见问题解决

### 程序不输出任何内容

检查：
- 是否有 `main` 函数
- `main` 函数是否有正确的缩进
- 是否调用了 `out` 函数

### 文件操作失败

检查：
- 文件路径是否正确
- 是否有写入权限
- 文件是否已被其他程序占用

### AI 调用返回错误

检查：
- `OPENAI_API_KEY` 环境变量是否设置
- 网络连接是否正常
- 模型名称是否正确
- API 配额是否充足

### 程序运行缓慢

检查：
- 是否有无限循环
- AI 调用是否过于频繁
- 是否在读取大文件
