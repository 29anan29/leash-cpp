## 1. 两种包的区别

Leash 世界里的「包」分为两类，它们**写法和引入方式完全不同**，先分清再说：

| 维度 | 原生包（Native Package） | 纯 Leash 包（Pure Package） |
|------|--------------------------|-----------------------------|
| 实现语言 | C++（位于 `src/host/native.cpp` 等） | Leash 自身（`.ae` 文件） |
| 引入指令 | `package 名称` | `import "名称"` |
| 典型例子 | `package file` / `package ai` / `package json` | `import "math"` / `import "string"` / `import "slug"` |
| 能力来源 | 宿主进程注入（Capabilities，如文件/网络/AI） | 普通函数被合并进同一份字节码 |
| 适用场景 | 必须访问宿主机（文件、进程、网络、随机数、时间、加密、AI、环境变量） | 纯算法、文本处理、集合操作、数据转换等任何不碰宿主机的逻辑 |
| 调试方式 | 需重新 `make` 编译器；用最小例子验证注入 | 改完即跑，无需重编译 |
| 跨平台 | 受 C++ 实现限制 | 纯逻辑，跨平台一致 |

**经验法则（请刻在脑子里）**：能用纯 Leash 写的，一律用纯 Leash 写。只有在「需要触碰宿主机」时才动用对应的原生包。原因有三：

1. 纯 Leash 包更易读、可跨平台、无需重新编译 C++；
2. 纯 Leash 包随时可改、可单测、可调试，不依赖编译器重建；
3. 纯 Leash 包天然遵循「最小特权」原则——它不会自带任何宿主能力，调用方能否 `out`/`read` 完全取决于调用方的上下文，安全性更可控。

> 一句话：**纯 Leash 包是「函数集合」；原生包是「能力集合」。**
> 你写给 Agent 用的工具，绝大多数应该是前者；只有当工具「必须做宿主机动作」时，才需要后者。

### 1.1 一个心智模型：包是「被拼进同一份字节码的函数表」

无论从 `import` 还是 `package` 来，最终所有函数都会进入**同一张全局函数表**，被编译进同一份字节码。区别只在于：

- `import "foo"` 是把 `lib/foo.ae` 这个**文本文件**的内容（一组 `fn`）合并进来；
- `package file` 是把 C++ 里 `makeFilePkg()` 这组**原生函数**按名字注入进来。

正因如此：

- 函数是「按名字」解析的——**重名会静默覆盖**（后 import 的赢），所以前缀纪律极其重要（见第 4.3 节）；
- 库函数不是「被调用的子进程」，而是「被内联进同一程序的函数」，所以「能力」也随调用上下文走（见第 3 节的能力模型）。

## 2. 用 C++ 写原生包（宿主能力）

> 本节面向**需要给 Leash 增加新宿主能力**的编译器开发者。绝大多数包作者**不需要**读这一节，
> 直接跳到第 3 节即可。但了解底层机制有助于你理解 `package` 指令到底做了什么。

当现有原生包（`file`/`os`/`http`/`crypto`/`ai`/`json`/`time`/`random`/`re`/`core`）都没覆盖你的需求时，
例如你想新增一个 `sqlite` 包、或一个 `clipboard` 包，才需要写 C++ 原生包。

### 2.1 核心概念

- 每个原生包是一个 `Package` 结构体：`{ name, funcs: [{name, arity, fn}] }`。
- `fn` 是 C++ 函数指针：`Value fn(const std::vector<Value>& args, HostContext& ctx)`。
- 所有包在 `initBuiltinPackages()` 里用 `registerPackage(makeXxxPkg())` 注册进全局表。
- 当源码出现 `package 名称` 时，编译器把该包名记录到 `prog.packages`；`main.cpp` 的
  `injectPackage` 按名字把包里的原生函数**注入**为 `isNative` 的全局函数；VM 调用时直接走
  `callee.nativeFn(args, ctx)`，不经过字节码。

**关键认知**：`package file` 只是「告诉编译器我需要 file 能力」；真正能用的是注入后那些
**裸函数名**（`read`/`write`/`exists`/`delete`/`append`），而不是 `file.read(...)`。

### 2.2 三步写一个新原生包

**第 1 步：在 `src/host/native.cpp` 实现函数**（放在 `namespace leash { namespace native {` 内）：

```cpp
Value native::mypkg_greet(const std::vector<Value>& args, HostContext& ctx) {
    std::string who = valueToStr(ctx.store(), args[0]);
    return Value::makeStr(ctx.store().intern("你好, " + who));
}
```

> 取值用 `valueToStr(ctx.store(), args[i])`；返回字符串用 `Value::makeStr(ctx.store().intern(s))`；
> 返回整数 `Value::makeInt(n)`、布尔 `Value::makeBool(b)`、空 `Value::nil()`、浮点 `Value::makeFloat(f)`。

**第 2 步：写 `makeXxxPkg()` 并在 `initBuiltinPackages()` 注册**：

```cpp
static Package makeMyPkg() {
    Package pkg{"mypkg", {
        {"greet", 1, native::mypkg_greet},
    }};
    return pkg;
}

void initBuiltinPackages() {
    // ...已有包...
    registerPackage(makeMyPkg());   // 新增此行
}
```

**第 3 步：在 Leash 源码里使用**：

```leash
package mypkg
fn main
    out(greet("Leash"))    // 注意：暴露的是裸名 greet，不是 mypkg.greet
```

### 2.3 参数与返回值约定

- `arity` 必须与实际参数个数一致；VM 会按 `arity` 收集实参。若你声明 arity=2 却只传 1 个，VM 会报错。
- 下标越界请自行 `throw std::runtime_error("...")`，会被宿主捕获并报告为运行时错误。
- 需要读宿主策略（如是否允许联网）时，用 `ctx` 上的接口或 `ctx.audit().log(...)` 记录审计日志。
- 返回容器（list/map）时，优先用 `Value::makeList` / `Value::makeMap` 的工厂函数，避免在栈上泄漏引用。

### 2.4 调试原生包

- 改完 `native.cpp` 后必须重新 `make`（整个编译器重建）。
- 若运行报「未知函数或标识符」，按顺序排查：
  1. `registerPackage` 是否真的被调用（拼写、是否在 `initBuiltinPackages` 内）；
  2. `package 名称` 是否写在**顶层**（不能在函数内、不能在缩进块内）；
  3. `arity` 是否与调用处实参数一致；
  4. 函数名是否跟其它已注册包的裸名冲突（冲突会静默覆盖）。
- 先用 `out(greet("x"))` 这种最小例子验证注入是否成功，再接复杂逻辑。

### 2.5 何时「不」该写原生包

- 纯算法、文本处理、集合操作：一律用纯 Leash 包（更易读、可跨平台、无需重编译）。
- 能复用现有原生包（`file`/`os`/`http`/`crypto`/`ai`/`json`/`time`/`random`/`re`/`core`）就复用，不要重复造轮子。
- 能用 `read` + 文件 + 纯逻辑实现的，就不要为了「快一点」去写 C++。`lib/` 里绝大多数包都是纯 Leash。

## 3. 一个最小纯 Leash 包（含运行验证）

纯 Leash 包的本质：**一个 `.ae` 文件 + 一组普通函数**。没有 `package` 块、没有 `export` 关键字、
没有命名空间语法——所有顶层 `fn` 在被 `import` 后都会变成调用方的全局函数。

新建 `lib/greet.ae`：

```leash
// greet 包 v1.0.0
// 导入后可直接调用：greet_hi / greet_bye
// 依赖：无（纯逻辑，不需要任何原生包）

fn greet_hi(name: str) -> str
    return "你好，" + name + "！"

fn greet_bye(name: str) -> str
    return "再见，" + name + "。"
```

使用方 `examples/use_greet.ae`：

```leash
import "greet"
fn main
    out(greet_hi("Leash"))
    out(greet_bye("Leash"))
```

运行（在项目根目录）：

```bash
LEASH_LIB=./lib ./leash examples/use_greet.ae
# 输出：
# 你好，Leash！
# 再见，Leash。
```

### 3.1 能力模型：为什么库函数不能 `out`

这是初学者最容易踩、也最关键的一条。把 `greet` 改写成「自己打印」会怎样？

```leash
// ❌ 错误示例：库函数里直接 out
// lib/greet.ae
fn greet_hi(name: str)
    out("你好，" + name + "！")   // 运行时报：函数 greet_hi 缺少能力: io
```

**原因**：`out` 与 `in` 是宿主注入的 **IO 能力**。当前 Leash 编译器只在 `fn main` 里开放这个能力。
库函数（非 main）没有被授予 `io` 能力，一旦调用 `out`/`in` 就会被能力检查拦下，报「缺少能力: io」。

这其实是一个**好消息**：它强制你把「计算」与「展示」分开。库函数只负责把值算出来并返回，
由调用方的 `main` 决定要不要打印、打印到哪。这种「纯函数 + main 负责副作用」的结构，
让库函数天然可测试、可组合、可复用——你不需要为了「验证它对不对」而真的去打印。

> 铁律：**库函数只算不打印，返回值交给 main。** 这一条贯穿整个教程，详见第 10 节坑位 2。

### 3.2 最小包的三个要点

- 包就是**一个 `.ae` 文件**，里面是一组普通函数。
- 函数名建议加**统一前缀**（如 `greet_`），避免与其它包的重名函数互相覆盖。
- 文件顶部写注释，说明「导入后可直接调用哪些函数」以及「依赖哪些包」。
- 库函数**只计算、不打印**：它返回 `str`，由调用方的 `fn main` 用 `out(...)` 打印。

## 4. 包的目录结构与命名约定

### 4.1 标准库目录 `lib/`

官方与社区包统一放在 `lib/` 下。当前仓库 `lib/` 里已有 **116 个**纯 Leash 包（`math`/`string`/`list`/
`dictx`/`slug`/`statsx`/`csvx`/`toml`/`yaml`/`httpx`/`promptx`/`embedx` 等）。你新建的包也放这里，
就能被任意 `import "你的包名"` 找到。

推荐的项目目录结构：

```
你的项目/
├── leash                  # 编译器可执行文件（make 产物）
├── lib/                   # 标准库与你的自建包
│   ├── math.ae            # 官方包
│   ├── string.ae
│   ├── slug.ae
│   └── mytool.ae          # ← 你新建的包
├── examples/              # 示例程序
│   └── use_mytool.ae
└── tests/                 # 你的包的单测
    └── test_mytool.ae
```

### 4.2 文件命名 = 包名

`lib/foo.ae` → `import "foo"`。约定：

- 文件名全部**小写**，多词用**连字符或下划线**均可（`slug.ae`、`csvx.ae`、`statsx.ae` 都是官方示例）。
- **不要**加扩展名风格如 `mytool.leash`——编译器只认 `.ae`，且 `import "mytool"` 会去找 `mytool.ae`。
- 一个文件 = 一个包（见第 20 节关于多文件包的说明）。

### 4.3 函数命名：统一前缀

Leash 没有命名空间，所有顶层函数被 import 后都进入**同一个全局命名空间**。因此：

- 包 `slug` 的函数统一叫 `slugify`（前缀 `slug_` 或 `slug` 即可，保持简洁）。
- 包 `string` 的函数统一叫 `str_upper` / `str_lower` ...（前缀 `str_`）。
- 包 `list` 的函数统一叫 `list_push` / `list_sum` ...（前缀 `list_`）。
- 包 `math` 的函数统一叫 `math_gcd` / `math_avg` ...（前缀 `math_`）。

你的包也请遵守：**所有导出函数以 `包名缩写_` 开头**。例如包 `mytool` 的导出函数就叫
`mytool_clean` / `mytool_count` ...。这样即便别人同时 import 了十个包，也不会出现 `clean`
这种通用名称满天飞、互相覆盖的惨剧。

> 反例：`fn clean(s: str)` 这种无前缀函数，一旦和别人的 `clean` 重名，后 import 的会覆盖先 import 的，
> 且**没有任何报错**——这是最隐蔽的 bug 来源之一。建议内部辅助函数也可用前缀，避免「私有限制」靠前缀约定实现。

### 4.4 前缀命名的两层结构

当你的包导出函数较多时，可以再用「子模块」式二级前缀，例如一个 `textx` 包：

- `textx_split_paragraph`
- `textx_count_words`
- `textx_extract_quotes`

这样即便函数很多，名字也不会和别的包打架，调用方也能从名字猜出函数归属。

## 5. 从零写一个纯 Leash 包的完整流程（五步法）

下面用「写一个把字符串按词频统计的包 `wordcount`」为例，走完整流程。这五步适用于任何纯 Leash 包。

### 第 1 步：想清楚「这个包解决什么问题、导出哪些函数」

不要上来就写代码。先用一句话定义包的职责边界：

> `wordcount` 包：给定一个文本字符串，统计每个词出现的次数，并提供「按次数排序取 Top-N」的辅助函数。

导出函数规划：

- `wordcount_count(text: str) -> map`：返回 `{词: 次数}` 的映射。
- `wordcount_top(m: map, n: int) -> list`：返回 `[ [词, 次数], ... ]`，按次数降序取前 n 个。

### 第 2 步：写包文件，库函数只算不打印

`lib/wordcount.ae`：

```leash
// wordcount 包 v1.0.0
// 导入后可直接调用：wordcount_count / wordcount_top
// 依赖：string（用于 str_lower / str_split / str_trim）

import "string"

fn wordcount_count(text: str) -> map
    let low = str_lower(text)
    let words = str_split(low, " ")
    let mut m = {}
    let mut i = 0
    while i < len(words)            // 用索引式 while 遍历 str_split 结果（坑位 19）
        let w = words[i]
        if len(w) > 0               // str_split("a  b"," ") 会产生空串，跳过即可
            let cur = json_get(m, w)
            if cur == nil
                m[w] = 1
            else
                m[w] = cur + 1
        i = i + 1
    return m

fn wordcount_top(m: map, n: int) -> list
    let keys = json_keys(m)
    let mut items = []
    let mut ki = 0
    while ki < len(keys)           // 遍历 map 用 json_keys + 索引 while（坑位 19）
        let k = keys[ki]
        items = items + [[k, json_get(m, k)]]
        ki = ki + 1
    let mut i = 0
    while i < len(items)
        let mut j = i + 1
        while j < len(items)
            if items[j][1] > items[i][1]
                let tmp = items[i]
                items[i] = items[j]
                items[j] = tmp
            j = j + 1
        i = i + 1
    if n > len(items)
        return items
    return items[0:n]
```

> ⚠️ 关于 `continue`：**在 Leash 的 `for ... in` 循环里 `continue` 会触发死循环（报「燃料耗尽」）**。
> 所以上面用 `if len(w) > 0 { ... }` 把「跳过空词」的语义表达成「条件成立才处理」，
> 而不是 `if 空 { continue }`。需要 `continue` 语义时，要么用 `if` 包住正常逻辑，要么改用 `while`（`while` 里的 `continue` 是安全的）。详见第 10 节坑位 1。
> 另外注意：`str_split("a  b", " ")` 会产生空串，这里用 `len(w) > 0` 直接跳过；若字符串可能含首尾空白，
> 可在「整段参数」上用 `str_trim`（如 `str_split(str_trim(s), " ")`），但**不要**在循环里对 split 出来的切片 `str_trim`（坑位 20）。

### 第 3 步：写使用示例，在 `fn main` 里调用并打印

`examples/use_wordcount.ae`：

```leash
import "wordcount"
import "string"
fn main
    let text = "apple banana apple cherry banana apple"
    let m = wordcount_count(text)
    out(stringify(m))
    let top = wordcount_top(m, 2)
    out(stringify(top))
```

运行：

```bash
LEASH_LIB=./lib ./leash examples/use_wordcount.ae
# 输出类似：
# {"banana": 2, "apple": 3, "cherry": 1}
# [["apple", 3], ["banana", 2]]

## 6. 包的命名与放置

- **文件名即包名**：`lib/math.ae` → `import "math"`；`lib/mycsv.ae` → `import "mycsv"`。
- 编译器按以下顺序查找裸模块名（任一命中即停）：
  1. 入口文件所在目录的 `lib/`
  2. 环境变量 `LEASH_LIB` 指向的目录
  3. 可执行文件所在目录的 `lib/`
  4. （历史上还支持 `/usr/local/lib/leash`，以实际编译配置为准）
  5. 当前工作目录的 `lib/`（即 `cwd/lib`，方便在仓库根直接 `./leash x.ae`）
- **包名用小写、无空格**；函数名 `包名_动作` 风格最稳妥，避免和内置类型/保留字撞车。
- 不要把包放进 `lib/` 的子目录再试图 `import "sub/foo"`——当前 `import` 只认单文件裸名，
  目录式包不被原生支持（见第 20 节绕行方案）。

### 6.1 一个常见误区：把包放进 `examples/lib`

很多新手在 `examples/` 下也建一个 `lib/`，以为「入口在 examples 下就会找 examples/lib」。这**确实有效**——
因为搜索路径第 1 条就是「入口文件所在目录的 `lib/`」。但这样会让同一个包在仓库里出现两份拷贝，容易版本漂移。
更推荐的做法是：包统一放仓库根 `lib/`，运行示例时用 `LEASH_LIB=./lib ./leash examples/x.ae`。

## 7. 依赖其他包（import / package）

纯 Leash 包可以依赖其它纯 Leash 包，也可以使用原生包的能力。两条铁律：

> **铁律一**：依赖**纯 Leash 包**用 `import "名称"`。
> **铁律二**：依赖**原生包**（需要宿主能力）用 `package 名称`。

### 7.1 依赖纯 Leash 包：用 `import`

```leash
// lib/setx.ae
import "list"          // 复用 list_index / list_contains

fn setx_has(xs: list, x) -> bool
    return list_index(xs, x) != -1

fn setx_add(xs: list, x) -> list
    if list_index(xs, x) == -1
        return xs + [x]
    return xs
```

### 7.2 依赖原生包：用 `package`

```leash
// lib/fsx.ae
package file          // 注入 read / write / exists / delete / append
import "string"       // 复用 str_split / str_join

fn fsx_read_lines(path: str) -> list
    return str_split(read(path), "\n")

fn fsx_write_lines(path: str, lines: list)
    write(path, str_join(lines, "\n"))
```

> 千万不要写成 `import "file"`——那会去 `lib/` 找 `file.ae`（不存在）而报错。原生包没有 `.ae` 文件，
> 它是 C++ 注入的，只能用 `package` 声明。

### 7.3 关于「传递依赖」的一个真相

由于编译器在 `assemble()` 阶段会**递归收集并去重**所有被 import 的文件，最终把它们编译进同一份
字节码，所以：**当你 `import "csvx"` 时，csvx 自己 `import "string"` 的那些函数（如 `str_split`）
也会进入你的全局命名空间**。

这意味着：

- 你 import 一个包后，**不必再重复 import 它的依赖**就能直接调用（例如 import csvx 后可直接用 `str_split`）。
- 但代价是全局命名空间会被「传染」——你本不想引入的 `str_*` 也进来了。若你恰好也想定义一个 `str_*`，
  就会冲突。所以**起名前缀依旧重要**。

### 7.4 循环依赖

`assemble()` 自带**环检测**。如果你让 A import B、B 又 import A，编译器会报错而不是死循环。
因此请保持包的依赖是**有向无环**的：底层包（string/list/math）不依赖上层业务包。

### 7.5 关于「值传递」的提醒

Leash 的 list / map 在函数调用时是**值语义**（传副本，或被 VM 以不可变方式处理）。这意味着：

- 你在库函数里 `m[k] = v` 修改的是一个**局部副本**，调用方传进来的原 map **不会**被改。
- 如果你想「就地修改」调用方的数据，正确的做法有两种：
  1. 函数返回新的 list / map，由调用方接住（`xs = mytool_append(xs, x)`）；
  2. 或让调用方把变量传进来、函数返回新值（最推荐，最符合「纯函数」风格）。

来看一个具体例子：

```leash
// ❌ 容易误以为会改外部变量
import "list"
fn bad_push(xs: list, x)
    xs = list_push(xs, x)     // 只改了局部 xs，调用方的 xs 不变

// ✅ 正确：返回新列表，调用方接住
import "list"
fn good_push(xs: list, x) -> list
    return list_push(xs, x)
```

## 8. 标准库搜索路径（import 怎么被找到）

为了让别人能 `import "你的包"`，把 `你的包.ae` 放进任一标准库目录即可。临时指定目录：

```bash
export LEASH_LIB=/path/to/your/lib
./leash app.ae
```

或在命令前内联：

```bash
LEASH_LIB=./lib ./leash examples/use_wordcount.ae
```

搜索顺序示意图：

```
import "foo"
   │
   ├─ 1. 入口文件所在目录/lib/foo.ae  ?
   ├─ 2. $LEASH_LIB/foo.ae            ?
   ├─ 3. 可执行文件目录/lib/foo.ae    ?
   ├─ 4. /usr/local/lib/leash/foo.ae  ?   (视编译配置)
   └─ 5. cwd/lib/foo.ae               ?
        │
        └─ 都找不到 → 报「无法解析的模块: foo」
```

调试「import 找不到」的排查清单：

1. 文件确实存在吗？`ls lib/你的包.ae`。
2. 包名拼写对吗？`import "slug"` 找的是 `slug.ae`，不是 `slugify.ae`。
3. `LEASH_LIB` 指向的目录对吗？用 `echo $LEASH_LIB` 确认。
4. 入口文件所在目录的 `lib/` 是否放对了？如果入口是 `examples/x.ae`，那 `examples/lib/` 也会被搜。
5. 文件名后缀是 `.ae` 吗？其它后缀一律不认。

## 9. 编写规范与风格约定

1. **函数加前缀**，避免全局命名冲突（`math_gcd`、`str_upper`、`slugify`）。
2. **`out`/`in` 是保留字**：内部变量改用 `res`/`buf`/`acc`/`inp`/`tmp`。
3. **类型名也是保留字**：`int`/`float`/`str`/`bool`/`list`/`map` 不能当变量名。
   （注意：原生包 `random` 提供的随机整数函数是裸名 `int(lo, hi)`，与类型 `int` 不冲突，但你自己
   不要声明一个叫 `int` 的变量去遮蔽它。）
4. **无高阶函数**：不要尝试把函数当参数传；需要「对集合映射」就老老实实写循环。
5. **JSON 文本里的 `{` 要转义**：在字符串里写 JSON 时，`{` 会被当作插值，需写成 `\{`；
   更简单的方式是**直接用映射字面量** `{}` / `{"k": v}`，根本不碰字符串里的花括号。
6. **`get` 是 HTTP 原生包占用的名字**：JSON 取值请用 `json_get`；你的包里不要定义名为 `get` 的函数，
   以免和 `http` 包的 `get(url)` 冲突。
7. **切片**：`xs[a:]` 合法；`xs[:b]` 请写成 `xs[0:b]`（下限不能省略）。
8. **没有三元 `?:`**：条件赋值用 `if/else` 写，绝不要写 `x = c ? a : b`。
9. **参数顺序**：无类型标注的参数必须排在有类型标注的参数之前，如 `fn f(x, y: int) -> int`。
10. **map 参数标注**：接收映射的参数标 `: map`，如 `fn f(m: map)`。
11. **缩进即语法**：代码块靠缩进（同 Python），没有 `{}`；缩进不一致会报「需要缩进块」。
12. **空 if 体不行**：`if` 体不能仅由注释构成，至少要有一条语句（或放一个占位表达式，但更推荐直接把逻辑写进去）。
13. **没有 `float()` 转换函数**：整数转浮点请用「除以 `2.0` 之类的小数」，例如 `(a + b) / 2.0`；
    不要写 `float(a)`（会报「未知函数或标识符」）。详见第 10 节坑位 0。
14. **整数除法会截断**：`5 / 2 == 2`（向下取整到整数）；想要小数结果，让任一操作数为浮点：`5 / 2.0 == 2.5`。

## 10. 已知坑位与最佳实践（必读 ⚠️）

这一节是**全教程价值最高**的部分。下面每一条都是从真实踩坑中总结的，覆盖了 Leash 当前编译器
「看起来合理、实际会炸」的所有地方。**请逐条对照你的代码。**

### 坑位 0：`float()` 不是内置函数，整数除法会截断

**现象**：
- 写 `let f = float(3)` 报「未知函数或标识符: float」。
- 写 `let mid = xs[len(xs)/2]` 以为拿到中点，结果在偶数长度时偏移一位（因为 `/` 是截断除法）。
- 写 `(a + b) / 2` 想算平均值，结果得到整数而非小数。

**原因**：Leash 当前**没有** `float()` 这种类型转换内置函数；`/` 在两个整数运算时执行**截断整数除法**。

**正确做法**：

```leash
// ❌ 错误：float() 不存在
let f = float(3)

// ✅ 正确：除以小数得到浮点
let f = 3 / 2.0          // 1.5
let avg = (a + b) / 2.0  // 浮点平均

// 中位数（偶数长度）取中间两数均值，用 /2.0 保精度
fn my_median(xs: list) -> float
    let s = list_sort(xs)
    let n = len(s)
    if n % 2 == 1
        return s[n / 2] / 1.0       // 单数：转成浮点返回
    let a = s[n / 2 - 1]
    let b = s[n / 2]
    return (a + b) / 2.0            // 双数：浮点均值
```

> 经验：凡是要「浮点结果」的地方，让至少一个操作数是 `.0` 小数。

### 坑位 1：`continue` 在 `for ... in` 循环里会死循环

**现象**：在 `for x in xs` 或 `for i, e in xs` 或 `for k, v in m` 里用 `continue`，程序跑着跑着
报「[运行时错误] 燃料耗尽」。

**原因**：当前 VM 对 `for ... in` 的 `continue` 实现有误，不会正确推进迭代器。

**正确做法**：用 `if 条件 { ... }` 反向表达「跳过」，或改用 `while` 循环（while 里的 continue 正常）。

```leash
// ❌ 错误：for 里用 continue 会死循环
for w in words
    if len(w) == 0
        continue
    // 处理 w

// ✅ 正确：用 if 包起来
for w in words
    if len(w) > 0
        // 处理 w

// ✅ 也正确：用 while（continue 在 while 里可用）
let mut i = 0
while i < len(words)
    let w = words[i]
    if len(w) == 0
        continue
    // 处理 w
    i = i + 1
```

> `break` 在 `for ... in` 里是**正常**的，可以放心用；只有 `continue` 有问题。

### 坑位 2：库函数里调用 `out` 会报「缺少能力: io」

**现象**：你在 `lib/xxx.ae` 的某个 `fn` 里写了 `out(...)`，结果运行报「函数 xxx 缺少能力: io」。

**原因**：`out`/`in` 是宿主注入的 IO 能力，当前**只有 `fn main` 内**才被允许。库函数（非 main）没有这个能力。

**正确做法**：库函数**只计算、返回值**；由调用方的 `fn main` 负责打印。

```leash
// ❌ 错误：库函数直接打印
fn greet(name: str)
    out("你好，" + name)

// ✅ 正确：库函数返回，main 打印
fn greet(name: str) -> str
    return "你好，" + name

// 使用方：
fn main
    out(greet("Leash"))
```

### 坑位 3：`out`/`in`/`match`/`cap` 是保留字，不能当变量名

**现象**：写 `let out = 1` 报「let 后需要变量名」；写 `let in = x` 同样报错。

**正确做法**：改用 `buf`/`res`/`acc`/`inp`/`tmp` 等普通名。

```leash
// ❌ 错误
let out = compute()        // 报：let 后需要变量名

// ✅ 正确
let res = compute()
```

### 坑位 4：类型名 `int`/`float`/`str`/`bool`/`list`/`map` 也不能当变量名

**现象**：`let str = "x"` 会报错（即便你只是想存个字符串）。

**正确做法**：变量名避开类型名，用 `s`/`text`/`buf` 等。

```leash
// ❌ 错误
let list = [1, 2, 3]

// ✅ 正确
let xs = [1, 2, 3]
```

### 坑位 5：字符串字面量里的 `{` 会被当作插值

**现象**：想打印一段含 `{` 的文本（比如 JSON 或模板），结果报「未定义变量」或插值错误。

**原因**：Leash 字符串支持 `"你 {x+1} 岁"` 这种插值（`{expr}` 内是任意表达式）。所以字面的 `{`
必须转义成 `\{`。

**正确做法**：字面花括号写 `\{`；或者，若是构造 JSON，直接用**映射字面量** `{}` 完全绕开字符串。

```leash
// ✅ 正确的插值用法
fn main
    let age = 18
    out("你 {age} 岁")          // 输出：你 18 岁

// ✅ 字面花括号要转义
fn main
    out("模板里有个字面括号: \{name}")   // 输出：模板里有个字面括号: {name}

// ✅ 构造 JSON 用映射字面量，别在字符串里拼花括号
fn main
    let m = {"name": "Leash", "age": 18}
    out(stringify(m))
```

> 注意：只有 `{` 需要转义；`}` 在字符串里是普通字符，不用转义。插值只对 `{...}` 整体生效。

### 坑位 6：字符串之间不支持关系比较

**现象**：写 `if c >= "0"` 或 `if s1 < s2` 报类型/运算符错误。

**原因**：Leash 不对字符串做字典序关系比较。

**正确做法**：先 `ord(c)` 转成整数再比较；或自己写逐字符比较函数。

```leash
// ❌ 错误：字符串不能 >= 比较
if c >= "0" and c <= "9"
    ...

// ✅ 正确：用 ord 转整数
let code = ord(c)
if code >= 48 and code <= 57        // '0'..'9'
    ...
```

### 坑位 7：映射赋值 `m[key] = v` 的 key 必须是简单变量或字符串字面量

**现象**：写 `m[a[i]] = v` 报「无法解析的表达式，遇到: =」。

**原因**：编译器解析 map 赋值左值时，只接受「简单变量名」或「字符串字面量」作为键。

**正确做法**：先把索引结果存进一个简单变量，再用它当键。

```leash
// ❌ 错误
let keys = ["a", "b"]
m[keys[0]] = 1        // 报：无法解析的表达式

// ✅ 正确
let keys = ["a", "b"]
let k = keys[0]
m[k] = 1
```

> 同样，`m[f(x)] = v`（`f(x)` 是调用结果）也不行，必须先把 `let tmp = f(x)` 再 `m[tmp] = v`。
> 读取时则灵活得多：`m[keys[0]]` 或 `m[f(x)]` 作为**右值**是允许的，只有「赋值左值」受限。

### 坑位 8：切片下限不能省略

**现象**：`xs[:3]` 报语法错误。

**正确做法**：下限写成 `0`，即 `xs[0:3]`；上限可省略（`xs[2:]` 合法）。

```leash
// ❌ 错误
let head = xs[:3]

// ✅ 正确
let head = xs[0:3]
let tail = xs[2:]        // 上限省略是允许的
```

### 坑位 9：没有三元运算符 `?:`

**现象**：写 `let y = c ? a : b` 报语法错误。

**正确做法**：用 `if/else` 赋值（Leash 支持 `if/else` 作为表达式返回，但仍推荐显式写法）。

```leash
// ❌ 错误
let label = n > 0 ? "正" : "非正"

// ✅ 正确
let mut label = "非正"
if n > 0
    label = "正"
```

### 坑位 10：函数参数顺序——无类型参数必须在前

**现象**：写 `fn f(y: int, x) -> int` 可能报参数声明错误。

**正确做法**：无类型标注的参数排前面，有类型标注的排后面。

```leash
// ✅ 正确
fn assert_eq(a, b, msg: str) -> bool
    return a == b

// ✅ 正确
fn list_get(xs: list, i: int)
    return xs[i]

// ❌ 容易出错的顺序（无类型参数在后）
fn f(y: int, x) -> int     // 不推荐，部分场景会报参数数量/类型问题
```

### 坑位 11：大循环里反复 `+` 拼接超长字符串，性能差

**现象**：循环上万次 `res = res + chunk`，越来越慢。

**正确做法**：能批量就批量；或把片段先收集到 `list`，最后 `str_join` 一次性拼。

```leash
// ❌ 慢：每次循环都重建整个字符串
let mut res = ""
for chunk in chunks
    res = res + chunk

// ✅ 快：先收集再 join
let mut parts = []
for chunk in chunks
    parts = parts + [chunk]
let res = str_join(parts, "")
```

### 坑位 12：不要定义名为 `get`/`post`/`now`/`sleep` 等的函数

这些名字被原生包（`http`/`time` 等）占用。若你 `package http` 又定义 `fn get(...)`，会冲突或遮蔽。
JSON 取值统一用 `json_get`，不要图省事叫 `get`。

### 坑位 13：`if` 体不能仅由注释构成

**现象**：写一个空 `if`（里面只有 `// 以后补`），报「需要缩进块 (INDENT)」。

**正确做法**：至少放一条语句；若真想占位，用 `let _placeholder = 0`（或后续直接补逻辑）。

```leash
// ❌ 错误
if debug
    // 暂未实现

// ✅ 正确
if debug
    out("debug 模式开启")
```

### 坑位 14：`for x in xs` 的 `x` 是「元素」不是「索引」

**现象**：想拿索引却只写了 `for x in xs`，拿不到下标。

**正确做法**：需要索引用 `for i, e in xs`（i 是索引，e 是元素）；只关心元素用 `for x in xs`。

```leash
// 遍历元素
for line in lines
    out(line)

// 遍历带索引
for i, line in lines
    out(i)
    out(line)
```

### 坑位 15：map 的遍历顺序不保证稳定

**现象**：两次运行同一段 `for k, v in m`，`out` 出来的键顺序可能不同。

**原因**：当前实现里 map 的底层顺序不承诺稳定（可能是哈希顺序）。

**正确做法**：**不要依赖 map 的遍历顺序**来做「排序输出」或「取第一个键」之类的逻辑。
需要有序时，把键收集到 list 再 `list_sort`：

```leash
fn sorted_keys(m: map) -> list
    let mut ks = []
    for k, v in m
        ks = ks + [k]
    return list_sort(ks)
```

### 坑位 16：`json_get` 取不到键时返回 `nil`，要与 `nil` 比较

**现象**：写 `if json_get(m, "x") == ""` 想判断「没有这个键」，却永远不成立（因为取不到返回的是 `nil` 不是空串）。

**正确做法**：用 `== nil` 判断缺失；要「带默认值」就显式判断后返回默认值。

```leash
// ✅ 正确：用 nil 判断
let v = json_get(m, "x")
if v == nil
    return "默认值"
return v
```

### 坑位 17：字符串索引返回的是「单字符字符串」，不是数字

**现象**：以为 `s[0:1]` 返回字符能直接当整数比，结果类型不对。要拿字符的 ASCII 码，用 `ord(s[0:1])`。

```leash
let c = s[0:1]        // 这是长度为 1 的字符串 "A"
let code = ord(c)     // 这是整数 65
```

### 坑位 18：递归有「燃料」上限

**现象**：写了一个上万层的递归（如超大数的斐波那契递归），跑到一半报「燃料耗尽」。

**正确做法**：递归适合几百层以内的场景（阶乘、树遍历、分治）；更深的用 `while` 迭代改写。
同时，能用迭代就别用递归做「线性累加」这类事。

```leash
// 阶乘：层数 = n，几百以内没问题
fn fact(n: int) -> int
    if n <= 1
        return 1
    return n * fact(n - 1)
```

### 坑位 19：`for ... in` 遍历 `str_split` 结果 + 循环体内有嵌套/拼接时，可能只迭代一次

**现象**：`for w in str_split(text, " ")` 并在循环体里调用 `str_trim(w)` 或做嵌套 `while`、向列表追加，
程序只处理了**第一个元素**就结束（或报「字符串索引越界」）。

**原因**：当前 VM 对「遍历 `str_split` 返回的列表」这种特定组合的处理存在缺陷——迭代器在第一次迭代后
就可能被内部状态破坏。同样的代码，如果数据源是「列表字面量」而非 `str_split` 结果，往往又是正常的，
这进一步说明问题是「遍历 split 结果」触发的。

**正确做法**：**凡是遍历 `str_split` 的结果，一律用索引式 `while` 循环**，不要 `for ... in`。这点和坑位 1
（for 里别用 continue）一脉相承——宁可多写几行 `while`，也不要赌 `for ... in` 的边角行为。

```leash
// ❌ 危险：for 遍历 str_split 结果 + 循环体内 str_trim / 嵌套
let words = str_split(s, " ")
for w in words
    if len(str_trim(w)) > 0
        ...

// ✅ 安全：索引式 while
let words = str_split(s, " ")
let mut i = 0
while i < len(words)
    if len(my_trim(words[i])) > 0
        ...
    i = i + 1
```

> 经验法则：**`str_split` 的结果，永远用 `while i < len(xs)` 遍历。** 普通列表（字面量、函数返回的列表）
> 用 `for ... in` 通常没问题，但涉及 split 就别用。

### 坑位 20：`str_trim` 作用在「含非 ASCII 字符的 `str_split` 切片」上，在循环内会崩溃

**现象**：在 `while` 循环里对 `str_split` 出来的某一行（含中文/全角标点）调用 `str_trim(line)`，
报「字符串索引越界」；但把同一行作为**字符串字面量**单独 `str_trim` 却正常。

**原因**：`str_trim` 在处理「引用了原字符串某一段缓冲区、且含多字节字符」的切片时，存在字节边界处理缺陷。
纯 ASCII 的切片一般没事，一旦切片里混了中文/全角字符就容易触发。

**正确做法**：在「遍历 split 结果」的场景下，**不要直接 `str_trim` 这些切片**。两种替代：
1. 自己写一个「只去 ASCII 空白」的字节安全 trim 助手（见下方 `my_trim`），它对内部多字节字符零侵入；
2. 或只在「整段字符串字面量/函数参数」上用 `str_trim`，绝不对 split 出来的行用。

```leash
// ✅ 字节安全 trim：只去掉空格(32)/制表(9)/换行(10)/回车(13)，不碰内部字符
fn my_trim(s: str) -> str
    let n = len(s)
    let mut a = 0
    while a < n
        let c = ord(s[a:a + 1])
        if c == 32 or c == 9 or c == 10 or c == 13
            a = a + 1
        else
            break
    let mut b = n
    while b > a
        let c = ord(s[b - 1:b])
        if c == 32 or c == 9 or c == 10 or c == 13
            b = b - 1
        else
            break
    return s[a:b]
```

> 提示：本教程第 27 节的 `mdtool` 包就内置了 `mdtool_trim`（同上思路）来规避该问题，可参考。

### 最佳实践小结（把上面的「正确做法」内化成习惯）

- 库函数：**只算不打印**，返回结果交给 main。
- 变量名：避开 `out`/`in`/`match`/`cap` 及所有类型名。
- 循环：`for` 里别用 `continue`，用 `if` 包逻辑；`break` 随意。
- 字符串：`{` 要 `\{`；关系比较先用 `ord()`。
- 映射键：先存简单变量再当键。
- 切片：下限写 `0`。
- JSON：优先映射字面量，别在字符串里拼花括号。
- 起名：统一前缀，避开 `get`/`post`/`now`/`sleep` 等原生占用名。
- 类型：`float()` 不存在，用 `/2.0`；整数除法会截断，要小数就带 `.0`。
- 顺序：map 遍历顺序不稳定，排序请走 list。

## 11. 如何依赖原生包（package 指令）

当你需要宿主能力时，在包文件**顶层**写 `package 名称`。可同时声明多个：

```leash
// lib/mycache.ae
package file          // 注入 read/write/exists/delete/append
package json          // 注入 parse/stringify/json_get/json_set/json_keys
package time          // 注入 now/sleep

fn mycache_put(path: str, key: str, value: str)
    let data = {}
    if exists(path)
        data = parse(read(path))
    write(path, stringify(json_set(data, key, value)))
```

原生包对照表（裸名直接调用）：

| package | 注入的裸函数 | 说明 |
|---------|--------------|------|
| `core` | `sqrt`/`pow`/`floor`/`ceil`/`round`/`abs` 等 | **自动注入**，无需声明 |
| `json` | `parse`/`stringify`/`json_get`/`json_set`/`json_keys` | 自动注入，也可显式声明 |
| `file` | `read`/`write`/`exists`/`delete`/`append` | 文件系统 |
| `os` | `env(name)`/`cwd()`/`exit()` | 环境变量/工作目录/退出 |
| `ai` | `chat(prompt)` | 无 key 时返回 mock |
| `time` | `now()`/`sleep(secs)` | 时间戳/休眠 |
| `random` | `int(lo,hi)`/`float()`/`choice(xs)` | 随机（注意：`random` 提供的 `float()` 与原生类型无关，是返回随机浮点的函数） |
| `crypto` | `sha256(s)` | 哈希 |
| `http` | `get(url)`/`post(url, body)` | 网络 |
| `re` | `ismatch`/`find` | 正则（避免 `{n}` 量词） |

> 注意：`core` 和 `json` 是**自动注入**的，即使不写 `package core` 也能用 `abs`/`parse` 等；
> 但显式写 `package json` 也能跑（幂等）。其余原生包必须显式 `package`。

> ⚠️ 关于 `random` 包：它注入的 `float()` 是一个**返回随机浮点数的函数**（不是类型转换！），
> 与第 10 节坑位 0 说的「没有 `float()` 类型转换内置函数」不矛盾——那个 `float()` 只在你
> `package random` 之后才作为「随机浮点」存在。为避免混淆，做类型转换时**永远用 `/2.0`**，
> 不要依赖 `random.float()`。

### 一个结合多个原生包的例子（计时装饰器风格）

```leash
package time
package random

fn bench_toss() -> int
    let t0 = now()
    let mut total = 0
    let mut i = 0
    while i < 1000
        total = total + int(1, 6)     // 模拟掷骰子 1000 次
        i = i + 1
    let t1 = now()
    return total + (t1 - t0)          // 把「结果+耗时」合成一个数返回给 main
```

> 这里故意**没有**在库函数里 `out`（坑位 2）。耗时信息应通过返回值带给调用方，由 main 决定怎么展示。
> 再次提醒：**库函数只算不打印**。

## 12. 如何组合多个 lib 包（分层与复用）

好的包应该是**薄而专**的：一个包只解决一类问题，然后通过 `import` 组合成更大的能力。

示例：做一个「配置读写」包，组合 `file` + `json`（其实 `lib/config.ae` 就是这么做的）：

```leash
// lib/configx.ae
package file
package json

fn configx_load(path: str) -> map
    if not exists(path)
        return {}
    return parse(read(path))

fn configx_save(path: str, cfg: map)
    write(path, stringify(cfg))

fn configx_get(cfg: map, key: str, default: str) -> str
    let v = json_get(cfg, key)
    if v == nil
        return default
    return v
```

再做一个「日志」包，组合 `time` + `file` + `string`：

```leash
// lib/logx.ae
package file
package time
import "string"

fn logx_write(path: str, level: str, msg: str)
    let line = str_join([stringify(now()), level, msg], " ") + "\n"
    append(path, line)
```

> 注意 `append` 是 `file` 包提供的（双参数：`append(path, content)`），不是 `list` 的 `list_push`。
> 别混淆「往文件末尾追加」和「往列表末尾加元素」。

### 分层原则

- 底层：纯逻辑包（`string`/`list`/`math`/`dictx`），不碰宿主，最稳定。
- 中层：IO 包装包（`fsx`/`configx`/`logx`），用 `package file` 等提供便捷封装。
- 上层：业务包（`validator`/`formatter`/`scheduler`），组合中层与底层。

依赖方向永远**向下**，不反向，避免循环依赖（第 7.4 节）。一个直观的依赖图：

```
业务包 (validator/formatter/scheduler)
        │ import
        ▼
中层包 (configx/logx/fsx)   ── package file/time/json
        │ import
        ▼
底层包 (string/list/math/dictx)   ← 纯净，无宿主依赖
```

## 13. 实战一：写一个字符串工具包 `strutil`

目标：补充标准库 `string` 没有的常用函数——`strutil_word_count`（统计词数）、
`strutil_title`（首字母大写）、`strutil_truncate`（按宽度截断并加省略号）、
`strutil_reverse_words`（反转词序）、`strutil_pad_center`（居中填充）。

`lib/strutil.ae`：

```leash
// strutil 包 v1.0.0
// 导入后可直接调用：strutil_word_count / strutil_title / strutil_truncate
//                   strutil_reverse_words / strutil_pad_center
// 依赖：string

import "string"

fn strutil_word_count(s: str) -> int
    let words = str_split(str_trim(s), " ")
    let mut n = 0
    let mut i = 0
    while i < len(words)                // 索引式 while 遍历 split 结果（坑位 19）
        if len(words[i]) > 0            // 多空格产生的空串直接跳过（坑位 20：别在循环里 str_trim 切片）
            n = n + 1
        i = i + 1
    return n

fn strutil_title(s: str) -> str
    if len(s) == 0
        return ""
    let first = str_upper(s[0:1])
    return first + s[1:len(s)]

fn strutil_truncate(s: str, width: int) -> str
    if len(s) <= width
        return s
    if width <= 1
        return s[0:width]
    return s[0:width - 1] + "…"

fn strutil_reverse_words(s: str) -> str
    let words = str_split(s, " ")
    let mut out_words = []
    let mut i = len(words) - 1
    while i >= 0
        if len(words[i]) > 0            // 用 len>0 跳过空串，避免循环内 str_trim 切片（坑位 20）
            out_words = out_words + [words[i]]
        i = i - 1
    return str_join(out_words, " ")

fn strutil_pad_center(s: str, width: int, ch: str) -> str
    let n = len(s)
    if n >= width
        return s
    let total = width - n
    let left = total / 2
    let right = total - left
    let mut pad = ""
    let mut i = 0
    while i < left
        pad = pad + ch
        i = i + 1
    let mut pad2 = ""
    let mut j = 0
    while j < right
        pad2 = pad2 + ch
        j = j + 1
    return pad + s + pad2
```

使用示例 `examples/use_strutil.ae`：

```leash
import "strutil"
fn main
    out(strutil_word_count("hello world  from leash"))
    out(strutil_title("hello"))
    out(strutil_truncate("这是一段很长很长的文本", 6))
    out(strutil_reverse_words("the quick brown fox"))
    out(strutil_pad_center("hi", 6, "*"))
```

运行：

```bash
LEASH_LIB=./lib ./leash examples/use_strutil.ae
# 输出：
# 4
# Hello
# 这是一段很长…
# fox brown quick the
# **hi**
```

> 注意 `strutil_truncate` 里 `s[0:width-1]` 的下限是 `0`（坑位 8），上限用算术表达式 `width - 1`，合法。
> 末尾用了中文省略号 `…` 而非三个点，纯属风格选择，不影响运行。
> `strutil_pad_center` 用两个 `while` 拼填充串，避免了「反复 `+` 拼接长串」的性能问题（坑位 11 的思路）。

## 14. 实战二：写一个统计工具包 `statsx`

目标：对一组整数做描述性统计。我们复用 `math` 包里已有的 `math_avg`（返回 float）来算均值，
自己实现中位数、众数、极差、标准差。

> ⚠️ 修复点：本教程早期版本用过 `float(...)` 做类型转换，但 Leash **没有** `float()` 内置函数，
> 那样写会报「未知函数或标识符」。下面所有「转浮点」都用 `/2.0` 或 `/1.0` 实现。

`lib/statsx.ae`：

```leash
// statsx 包 v1.0.0
// 导入后可直接调用：statsx_median / statsx_mode / statsx_range
//                   statsx_std / statsx_describe
// 依赖：math（math_avg）、list（list_sort / list_count）

import "math"
import "list"

fn statsx_median(xs: list) -> float
    let s = list_sort(xs)
    let n = len(s)
    if n == 0
        return 0.0
    if n % 2 == 1
        return s[n / 2] / 1.0
    let a = s[n / 2 - 1]
    let b = s[n / 2]
    return (a + b) / 2.0

fn statsx_mode(xs: list)
    if len(xs) == 0
        return nil
    let mut best = xs[0]
    let mut best_n = list_count(xs, xs[0])
    for e in xs
        let c = list_count(xs, e)
        if c > best_n
            best = e
            best_n = c
    return best

fn statsx_range(xs: list) -> int
    let s = list_sort(xs)
    return s[len(s) - 1] - s[0]

fn statsx_std(xs: list) -> float
    let avg = math_avg(xs)
    let mut acc = 0.0
    let mut i = 0
    while i < len(xs)
        let d = xs[i] - avg
        acc = acc + d * d
        i = i + 1
    return (acc / len(xs)) / 1.0

fn statsx_describe(xs: list) -> str
    let med = statsx_median(xs)
    let rng = statsx_range(xs)
    let avg = math_avg(xs)
    let sd = statsx_std(xs)
    return "均值=" + avg + " 中位数=" + med + " 极差=" + rng + " 标准差=" + sd
```

使用示例：

```leash
import "statsx"
import "math"
fn main
    let data = [3, 1, 4, 1, 5, 9, 2, 6]
    out(statsx_describe(data))
    out(statsx_mode(data))
```

> 说明：`math_avg` 返回 `float`，直接拼进字符串会自动转文本；`statsx_median` 用
> `(a + b) / 2.0` 做浮点除法，避免整数除法丢精度。这些都在真实环境验证过。

## 15. 实战三：写一个 CSV 读写包 `mycsv`

虽然官方已有 `csvx`（支持引号包裹、字段内逗号等），这里我们从零写一个**简化版**，
用于演示「解析—建模—序列化」的完整思路，并展示纯 Leash 如何处理文本分列。

`lib/mycsv.ae`：

```leash
// mycsv 包 v1.0.0（简化版 CSV，逗号分隔、无引号转义）
// 导入后可直接调用：mycsv_parse / mycsv_dump
// 依赖：string（str_split / str_join）

import "string"

fn mycsv_parse(text: str) -> list
    let lines = str_split(text, "\n")
    let mut rows = []
    let mut i = 0
    while i < len(lines)            // 索引式 while 遍历 split 结果（坑位 19）
        let line = lines[i]
        if len(line) > 0            // 跳过空行（坑位 20：循环内别 str_trim 切片）
            rows = rows + [str_split(line, ",")]
        i = i + 1
    return rows

fn mycsv_dump(rows: list) -> str
    let mut out_rows = []
    let mut i = 0
    while i < len(rows)
        out_rows = out_rows + [str_join(rows[i], ",")]
        i = i + 1
    return str_join(out_rows, "\n")
```

使用示例（含从文件读、处理后写回）：

```leash
package file
import "mycsv"
import "string"

fn main
    let text = "name,age\nAlice,30\nBob,25"
    let rows = mycsv_parse(text)
    out(mycsv_dump(rows))
    out("行数: " + (len(rows)))
    // 取每一行的第 0 列（名字）
    let mut names = []
    for row in rows
        names = names + [row[0]]
    out(str_join(names, " | "))
```

> 如果你的 CSV 需要引号包裹、字段内含逗号，请直接用官方 `csvx`（`csvx_read`/`csvx_write`），
> 它已经正确处理了 `"a,b"` 与 `""` 转义。这里是教学版。

## 16. 实战四：`cache` 包（基于 file + json）

目标：用文件实现一个简单的键值缓存（官方 `lib/cache.ae` 做了带 TTL 的版本，这里写一个更朴素的）。

```leash
// lib/simplecache.ae
package file
package json

fn simplecache_put(path: str, key: str, value: str)
    let data = {}
    if exists(path)
        data = parse(read(path))
    write(path, stringify(json_set(data, key, value)))

fn simplecache_get(path: str, key: str) -> str
    if not exists(path)
        return ""
    let data = parse(read(path))
    let v = json_get(data, key)
    if v == nil
        return ""
    return v
```

使用示例：

```leash
package file
import "simplecache"

fn main

```

> 首次写入时用映射字面量 `{}` 建空映射，避免「字符串里写 `{`」的转义麻烦（坑位 5）。

## 17. 实战五：`validator` / `formatter` / `scheduler`

### 17.1 validator（输入校验）

```leash
// lib/validator.ae
package re

fn validator_email(s: str) -> bool
    return ismatch("^[^@\\s]+@[^@\\s]+\\.[^@\\s]+$", s)

fn validator_phone_cn(s: str) -> bool
    // 注意：re 包的 {n} 量词支持不完善，这里用逐个字符校验更稳
    if len(s) != 11
        return false
    if s[0:1] != "1"
        return false
    let mut i = 1
    while i < 11
        let c = ord(s[i])
        if c < 48 or c > 57
            return false
        i = i + 1
    return true

fn validator_port(n: int) -> bool
    return n >= 1 and n <= 65535

fn validator_nonempty(s: str) -> bool
    return len(s) > 0
```

> 手机号校验故意**不用**正则 `{11}`（坑位：re 包 `{n}` 量词支持不完善），改用 `len` + `ord` 逐字符判断，
> 这样在任何环境都能跑通。邮箱校验用的正则不含 `{n}`，安全。

### 17.2 formatter（格式化输出）

```leash
// lib/formatter.ae
import "list"
import "string"

fn formatter_kv(pairs: list) -> str
    let mut res = ""
    let mut i = 0
    while i < len(pairs)
        let kv = pairs[i]
        res = res + kv[0] + " = " + kv[1] + "\n"
        i = i + 1
    return res

fn formatter_table(rows: list) -> str
    let mut res = ""
    let mut i = 0
    while i < len(rows)
        res = res + list_join(rows[i], " | ") + "\n"
        i = i + 1
    return res
```

> `pairs[i]` 假设每个元素是二元列表 `[k, v]`。注意 `kv[0]`/`kv[1]` 是列表下标访问，合法。

### 17.3 scheduler（基于 time）

```leash
// lib/scheduler.ae
package time

// 返回每次执行耗时的列表，由调用方 main 决定如何展示（库函数不 out）
fn scheduler_every(task: str, times: int, secs: int) -> list
    let mut durations = []
    let mut i = 0
    while i < times
        let t0 = now()
        // 这里只是示例「任务」：拼一条日志串
        let _log = task + " #" + i
        let t1 = now()
        durations = durations + [(t1 - t0)]
        sleep(secs)
        i = i + 1
    return durations
```

> 这个包把 `out` **移出了库函数**：`scheduler_every` 返回每次执行的耗时列表，由 main 负责打印。
> 这避免了在严格能力检查下「库函数缺 io」的报错，更符合「纯函数」约定（第 10 节坑位 2）。

## 18. 测试你的包（用 assert / assertx）

纯 Leash 包应当可单测。官方提供了两个断言包：

- `lib/assert.ae`：`assert_eq(a, b, msg)` / `assert_true(a, msg)`——**只比较、返回 bool，不打印、不中断**。
- `lib/assertx.ae`：`assertx_neq` / `assertx_contains` / `assertx_false`——扩展断言。

### 18.1 基本断言

```leash
// tests/test_slug.ae
import "slug"
import "assert"
fn main
    assert_eq(slugify("Hello, World!"), "hello-world", "基本 slug")
    assert_eq(slugify("  Trim Me  "), "trim-me", "去空格")
    assert_eq(slugify("a@b#c"), "a-b-c", "符号变连字符")
    out("slug 测试全部通过")
```

运行：

```bash
LEASH_LIB=./lib ./leash tests/test_slug.ae
```

> ⚠️ 重要事实：`assert_eq` / `assert_true` **失败时不打印、不中断、只返回 `false`**。
> 它不会因为你断言失败就停下来。所以如果你想要「首次失败立即终止」，就包一层 `if`：

```leash
import "assert"
fn main
    if not assert_eq(a, b, "msg")
        out("测试失败，提前退出")
        return
    out("继续其它测试")
```

### 18.2 用 assertx 做包含/不等断言

```leash
import "list"
import "assertx"
fn main
    assertx_contains([1, 2, 3], 2, "列表应包含 2")
    assertx_neq(1, 2, "1 不等于 2")
    assertx_false(false, "应为假")
    out("assertx 测试通过")
```

### 18.3 写一个「测试运行器」模式

因为 assert 不中断，建议你养成「最后汇总」的习惯：收集失败数，结尾统一报告。

```leash
import "assert"
fn main
    let mut failed = 0
    if not assert_eq(1 + 1, 2, "加法")
        failed = failed + 1
    if not assert_eq(len([1, 2]), 2, "长度")
        failed = failed + 1
    if failed == 0
        out("全部通过 ✅")
    else
        out("失败数: " + failed)
```

### 18.4 测试覆盖建议

每个包都建议附带 `tests/test_xxx.ae`，覆盖：

- **正常输入**：典型用例。
- **边界输入**：空串 `""`、空列表 `[]`、单元素、最大/最小值。
- **特殊字符**：中文、符号、空白、Unicode。
- **错误容忍**：传入明显非法值时的行为（返回默认值还是报错，要在文档里写清）。

示例（覆盖 wordcount 的边界）：

```leash
import "wordcount"
import "assert"
import "json"
fn main
    // 空串
    let m0 = wordcount_count("")
    assert_eq(len(m0), 0, "空串得到空映射")
    // 单重复词
    let m1 = wordcount_count("hi hi hi")
    assert_eq(json_get(m1, "hi"), 3, "hi 出现 3 次")
    // Top-N 超出总数
    let top = wordcount_top(m1, 10)
    assert_eq(len(top), 1, "Top10 实际只有 1 种词")
    out("wordcount 边界测试通过")
```

## 19. 调试技巧

- **打印中间值**：在可疑行前后用 `out(...)` 打印（仅限 main 内；库函数里用返回值传递）。
- **缩小复现**：把报错的包单独 `import` 跑最小用例，排除干扰。
- **看错误行**：编译器报错带「无法识别的字符 / 需要缩进块 / 参数数量不符 / 无法解析的表达式」等，
  对照第 10 节「已知坑位」逐条排查。
- **变量名冲突**：若报「未知函数或标识符」，多半是①忘记 `import` 依赖包；②函数名前缀拼错；
  ③`package` 漏写（需要原生能力却没声明）；④变量名用了保留字（坑位 3/4）。
- **JSON 转义**：字符串里的 `{` 引发插值错误，用 `\{` 或改用映射字面量（坑位 5）。
- **死循环/燃料耗尽**：优先怀疑 `for ... in` 里的 `continue`（坑位 1），或 `while` 里忘记 `i = i + 1`。
- **能力缺失**：报「缺少能力: io」说明在库函数里用了 `out`/`in`（坑位 2）。
- **类型问题**：报「运算符/类型」错误时，检查是否误把字符串当数字比（坑位 6），或忘了 `/2.0` 转浮点（坑位 0）。

> 调试利器：把 `LEASH_LIB=./lib` 固定写进 shell 的 `.env` 或 Makefile，避免每次手敲。

### 19.1 一个排查「未知函数」的清单

```
报错「未知函数或标识符: xxx」
  │
  ├─ xxx 是原生能力 (read/write/now/...) ?
  │     └─ 是否写了 package 对应包？（如 package file / package time）
  │
  ├─ xxx 来自别的纯 Leash 包 (str_split/list_sort/...) ?
  │     └─ 是否 import 了那个包？或 import 了依赖它的包（传递依赖）？
  │
  ├─ xxx 是你自己定义的函数？
  │     └─ 拼写对吗？前缀对吗？(setx_add vs setx_ad)
  │
  └─ xxx 是变量名被当成函数调用？
        └─ 检查是否漏写运算符，或把变量当函数用
```

## 20. 多文件包与项目组织

Leash 的 `import` 是按「裸名」在标准库路径找**单文件**的，当前**不直接支持「目录包」
（即 `lib/mypkg/` 这种带 `__init__` 的目录结构）**。推荐做法：

### 20.1 单文件包（首选）

所有函数放一个 `.ae`，靠前缀区分。绝大多数官方包都是单文件（如 `string.ae` 142 行、`list.ae` 116 行）。

### 20.2 必须拆分时：内部文件也放 `lib/` 并各自带前缀

如果你一个包实在太大，可以拆成多个文件，但每个内部文件**都要放在 `lib/`**，由主文件 `import` 它们，
且内部文件的函数也要带前缀（避免和别的包撞名）。

```
lib/
├── bigpkg.ae        // 主入口：import "bigpkg_core"; 对外总入口
└── bigpkg_core.ae   // 内部实现：函数加 bigpkg_ 前缀
```

`bigpkg.ae`：

```leash
import "bigpkg_core"
// 这里可以再包裹一层对外 API，或直接复用 bigpkg_core_* 函数
```

`bigpkg_core.ae`：

```leash
// 内部实现，不对外承诺稳定
fn bigpkg_core_helper(x: int) -> int
    return x * 2
```

> 注意：因为 `assemble()` 会递归收集，**主文件 import 后，bigpkg_core 的所有函数也进入全局命名空间**，
> 调用方其实能直接调 `bigpkg_core_helper`。如果你想「私有化」，目前没有语言级 private 机制，
> 只能靠**命名约定**（加 `_` 前缀表示内部、文档声明「不保证稳定」）来约束。

### 20.3 业务项目的多文件组织

对于一个完整业务程序（不是库），可以这样组织：

```
project/
├── main.ae              // 入口，import 各模块
├── lib/                 # 或把自建包放这里，用 LEASH_LIB 指向
│   ├── model.ae
│   └── service.ae
└── tests/
```

入口 `main.ae` 用 `import "model"` / `import "service"` 组装，标准库搜索路径会去 `cwd/lib` 找。

## 21. 性能与可读性

### 21.1 字符串拼接

见坑位 11：超大循环里别反复 `res = res + chunk`。先用 `list` 收集，最后 `str_join`。

### 21.2 列表追加

Leash 的列表是不可变语义的「拼接」（`xs + [x]` 返回新列表）。高频追加用 `while` + 索引累积，
或接受 `list_push` 的返回值。少量追加无所谓。

```leash
// 高频追加推荐写法：用 while + 索引，避免反复重建大列表
fn collect_evens(xs: list) -> list
    let mut res = []
    let mut i = 0
    while i < len(xs)
        if xs[i] % 2 == 0
            res = res + [xs[i]]
        i = i + 1
    return res
```

### 21.3 递归

Leash **支持递归**（如 `math_fib`、`math` 里的阶乘示例、目录树遍历）。递归写法清晰，但要注意
燃料（fuel）限制——过深的递归可能触发「燃料耗尽」。一般几百层的递归没问题，上万层需改迭代。

```leash
// 递归计算阶乘（仅演示，实际可用循环）
fn fact(n: int) -> int
    if n <= 1
        return 1
    return n * fact(n - 1)
```

### 21.4 循环里的重复计算

`len(xs)` 每次都会重新计算。在循环外层先存好：

```leash
// ❌ 每次都算 len
let mut i = 0
while i < len(xs)
    ...
    i = i + 1

// ✅ 提到外层
let n = len(xs)
let mut i = 0
while i < n
    ...
    i = i + 1
```

### 21.5 可读性

- 函数不超过 30~40 行，过长就拆。
- 一个函数只做一件事。
- 用清晰的变量名（`res`/`acc`/`tmp`/`idx`）而非 `a`/`b`/`x1`。
- 复杂分支加 `//` 注释说明「为什么」。
- 循环里避免重复计算（如 `len(xs)` 每次都算，可提到循环外 `let n = len(xs)`）。

## 22. 文档注释风格

Leash 只有 `//` 行注释（**没有** `#` 注释，`#` 会被当成语法错误）。推荐在包文件顶部写「三段式」文档：

```leash
// 包名 包 v版本号
// 一句话职责说明
// 导入后可直接调用：func_a / func_b / func_c
// 依赖：string / list（如果有）
//
// 设计说明（可选）：为什么这样实现、已知限制、性能特征。
```

函数级注释：

```leash
// slugify: 把任意标题转成 URL 友好的 slug。
//   规则：小写；字母数字保留；空格/连字符/下划线归一为 '-'；折叠连续 '-'；去首尾 '-'。
//   参数 s: 原始字符串
//   返回: slug 字符串
fn slugify(s: str) -> str
    ...
```

这样别人（和你三个月后的自己）扫一眼顶部就知道怎么用、依赖谁、有哪些坑。

### 22.1 给「会出错」的函数写「契约注释」

如果某个函数遇到非法输入会返回 `nil` 或抛错，请在注释里写清「前置条件」和「返回语义」：

```leash
// statsx_median: 返回中位数。
//   前置：xs 为数字列表；空列表返回 0.0。
//   返回：奇数长度取中点；偶数长度取中间两数均值（浮点）。
fn statsx_median(xs: list) -> float
    ...
```

## 23. 版本与依赖管理

Leash 目前**没有**官方的包版本/依赖解析机制（没有 `leash.toml`、没有 semver 锁文件）。我们用**约定**来管理：

- **版本写在顶部注释**：`// slug v1.2.0`。
- **破坏性变更改名前缀**：`slug_make` → `slug2_make`，旧名保留一段时间做兼容。
- **依赖用注释声明**：`// 依赖：string, list`，并在文件里实际 `import`。
- **复杂项目用 `LEASH_LIB` 指向自己的 lib 目录**，避免污染全局 `lib/`、避免拉到不兼容版本。
- **不要**在 `lib/` 里放「会改签名」的实验函数——一旦别人依赖，改签名就是破坏性变更。

> 未来若引入版本机制，很可能是「按目录分版本」或「包清单文件」，但当前请坚守上述约定。

## 26. 常见反模式清单

- ❌ 用 `out`/`in`/`match`/`cap` 或类型名作变量名（保留字，坑位 3/4）。
- ❌ 库函数里直接 `out`/`in`（缺 io 能力，坑位 2）。
- ❌ 在 `for ... in` 循环里用 `continue`（死循环，坑位 1）。
- ❌ 把函数当参数传（无 HOF，规范 4）。
- ❌ 纯 Leash 包里直接用 `read` 却不 `package file`（能力没注入）。
- ❌ 用 `import "file"` 代替 `package file`（去找不存在的 `.ae`）。
- ❌ 包名/函数名与已有包重名导致静默覆盖（第 4.3 节）。
- ❌ JSON 文本里忘写 `\{` 导致插值错误（或直接用映射字面量 `{}`）。
- ❌ 大循环里反复 `+` 拼接巨大字符串（性能差，坑位 11）。
- ❌ 写 `x = c ? a : b` 三元表达式（不支持，用 `if/else`）。
- ❌ 写 `m[a[i]] = v` 这种索引作 map 键（无法解析，坑位 7）。
- ❌ 写 `xs[:3]` 省略切片下限（语法错误，坑位 8）。
- ❌ 用 `c >= "0"` 做字符串关系比较（不支持，用 `ord()`）。
- ❌ 定义名为 `get`/`post`/`now`/`sleep` 的函数（原生占用，坑位 12）。
- ❌ 用 `float(...)` 做类型转换（不存在，坑位 0；改用 `/2.0`）。
- ❌ 假设 map 遍历顺序稳定（坑位 15；排序请走 list）。
- ❌ 用 `json_get(m, k) == ""` 判断「键不存在」（应 `== nil`，坑位 16）。
- ❌ 依赖递归做上万层线性累加（燃料耗尽，坑位 18；改用 while）。

## 27. 综合实战：构建一个 Markdown 工具包 `mdtool`

下面我们从**零到一**做一个完整、可发布、带测试的 Markdown 工具包 `mdtool`，
把前面所有知识串起来。目标功能：

1. `mdtool_headings(md: str) -> list`：提取所有标题行（`#`/`##`/...）。
2. `mdtool_toc(md: str) -> str`：生成目录（TOC）文本，带缩进。
3. `mdtool_strip_headings(md: str) -> str`：去掉标题的 `#` 符号，保留文字。
4. `mdtool_word_count(md: str) -> int`：统计正文词数（去掉标题符号后按空白分词）。
5. `mdtool_link_extract(md: str) -> list`：提取所有 `[text](url)` 链接的 `[url, text]` 列表。
6. `mdtool_code_blocks(md: str) -> list`：提取所有 ``` 围栏代码块的正文。
7. `mdtool_bold_extract(md: str) -> list`：提取所有 `**粗体**` 文本。

### 27.1 设计依赖

- 用 `string` 包做 `str_split`/`str_trim`/`str_join`/`str_contains`。
- 不依赖任何宿主能力（纯文本处理），所以**不需要 `package`**。

### 27.2 写包文件 `lib/mdtool.ae`

```leash
// mdtool 包 v1.0.0
// 导入后可直接调用：mdtool_headings / mdtool_toc / mdtool_strip_headings
//                   mdtool_word_count / mdtool_link_extract
//                   mdtool_code_blocks / mdtool_bold_extract
// 依赖：string
//
// 设计说明：以换行切分文本，逐行处理。标题以若干 '#' 开头。
// 链接提取用简单状态机扫描 '[' ']' '(' ')'，不依赖正则（避免 {n} 限制）。
// 代码块提取用 ``` 围栏配对；粗体提取用 ** ... ** 配对。
// 重要：所有「遍历 str_split 结果」都用索引式 while（坑位 19），并用字节安全的
//       mdtool_trim 代替 str_trim（坑位 20），以规避当前 VM 的已知问题。

import "string"

// 字节安全去空白：只去 ASCII 空白，不碰内部多字节字符（规避 str_trim 的 VM 问题）
fn mdtool_trim(s: str) -> str
    let n = len(s)
    let mut a = 0
    while a < n
        let c = ord(s[a:a + 1])
        if c == 32 or c == 9 or c == 10 or c == 13
            a = a + 1
        else
            break
    let mut b = n
    while b > a
        let c = ord(s[b - 1:b])
        if c == 32 or c == 9 or c == 10 or c == 13
            b = b - 1
        else
            break
    return s[a:b]

fn mdtool_headings(md: str) -> list
    let lines = str_split(md, "\n")
    let mut heads = []
    let mut i = 0
    while i < len(lines)
        let line = lines[i]
        if len(line) > 0
            if line[0:1] == "#"
                heads = heads + [line]
        i = i + 1
    return heads

fn mdtool_strip_headings(md: str) -> str
    let lines = str_split(md, "\n")
    let mut buf = []
    let mut i = 0
    while i < len(lines)
        let line = lines[i]
        if len(line) > 0 and line[0:1] == "#"
            let mut j = 0
            while j < len(line) and line[j:j + 1] == "#"
                j = j + 1
            buf = buf + [mdtool_trim(line[j:len(line)])]
        else
            buf = buf + [line]
        i = i + 1
    return str_join(buf, "\n")

fn mdtool_toc(md: str) -> str
    let lines = str_split(md, "\n")
    let mut toc = []
    let mut i = 0
    while i < len(lines)
        let line = lines[i]
        if len(line) > 0 and line[0:1] == "#"
            let mut j = 0
            while j < len(line) and line[j:j + 1] == "#"
                j = j + 1
            let level = j
            let title = mdtool_trim(line[j:len(line)])
            let mut indent = ""
            let mut k = 0
            while k < level - 1
                indent = indent + "  "
                k = k + 1
            toc = toc + [indent + "- " + title]
        i = i + 1
    return str_join(toc, "\n")

fn mdtool_word_count(md: str) -> int
    let plain = mdtool_strip_headings(md)
    let words = str_split(plain, " ")
    let mut n = 0
    let mut i = 0
    while i < len(words)
        if len(mdtool_trim(words[i])) > 0
            n = n + 1
        i = i + 1
    return n

fn mdtool_link_extract(md: str) -> list
    let mut links = []
    let mut i = 0
    let mut n = len(md)
    while i < n
        if md[i] == "["
            let mut j = i + 1
            let mut text = ""
            while j < n and md[j] != "]"
                text = text + md[j]
                j = j + 1
            if j < n and md[j] == "]"
                let mut k = j + 1
                if k < n and md[k] == "("
                    let mut p = k + 1
                    let mut url = ""
                    while p < n and md[p] != ")"
                        url = url + md[p]
                        p = p + 1
                    links = links + [[url, text]]
                    i = p + 1
                else
                    i = j + 1
            else
                i = i + 1
        else
            i = i + 1
    return links

fn mdtool_code_blocks(md: str) -> list
    let lines = str_split(md, "\n")
    let mut blocks = []
    let mut capturing = false
    let mut buf = []
    let mut i = 0
    while i < len(lines)
        let line = lines[i]
        if mdtool_trim(line) == "```"
            if capturing
                blocks = blocks + [str_join(buf, "\n")]
                buf = []
                capturing = false
            else
                capturing = true
        else
            if capturing
                buf = buf + [line]
        i = i + 1
    return blocks

fn mdtool_bold_extract(md: str) -> list
    let mut res = []
    let mut i = 0
    let mut n = len(md)
    while i < n - 1
        if md[i] == "*" and md[i + 1] == "*"
            let mut j = i + 2
            let mut text = ""
            while j < n - 1 and not (md[j] == "*" and md[j + 1] == "*")
                text = text + md[j]
                j = j + 1
            if j < n - 1
                res = res + [text]
                i = j + 2
            else
                i = i + 1
        else
            i = i + 1
    return res
```

> 注意点回顾：
> - 全程没有 `continue`（`for` 里用 `if len(...) > 0` 包逻辑，坑位 1）。
> - 没有 `out`（库函数只算不打印，坑位 2）。
> - `line[0:1]` 取首字符、`line[j:j+1]` 取单个字符，切片下限都是 `0`（坑位 8）。
> - 链接/粗体提取用**手写状态机**而非正则，避开 `re` 包的 `{n}` 限制，且更可控。
> - 变量名避开保留字与类型名（用 `n`/`j`/`k`/`p`/`text`/`url`，不用 `str`/`list`）。
> - `mdtool_code_blocks` 用 `mdtool_trim(line) == "```"` 判断围栏行（字节安全 trim，规避坑位 20）。

## 29. 包设计原则（SOLID 风格的朴素落地）

虽然 Leash 不是面向对象语言，但「写好一个包」的工程原则依然成立。下面把 SOLID 翻译成 Leash 包作者的朴素准则：

### 29.1 单一职责（SRP）

一个包只解决**一类**问题。`string` 管字符串、`list` 管列表、`math` 管数学。不要在一个包里
既做网络请求又做文本格式化——那会迫使调用方为了「格式化」而引入网络能力。

### 29.2 前缀即命名空间（替代 OCP/ISP）

没有接口/继承，我们用「前缀」模拟命名空间。前缀越具体，冲突面越小。例如 `strutil_` 比 `su_` 好，
因为前者自解释，后者容易和别人的 `su_` 撞。

### 29.3 依赖倒置（DIP）

上层业务包依赖**底层纯逻辑包**，而不是反过来。底层包绝不 `package file` 去碰宿主，这样底层包
可以被任何上下文（包括严格无 IO 的沙箱）复用。

### 29.4 可替换性

因为函数是「按名字」解析的，你的包被 `import` 后，调用方其实可以**自己定义一个同前缀函数**来「覆盖」你。
这既是风险（别人可能误覆盖），也是灵活性（测试时可 stub）。利用这一点，把「不确定、易变」的逻辑
做成独立的小函数，方便调用方替换。

### 29.5 一组「该做 / 不该做」

| 该做 | 不该做 |
|------|--------|
| 函数名统一前缀 | 用 `clean`/`get` 这种通用名 |
| 库函数只 return 值 | 库函数里 `out` |
| map 参数标 `: map` | 用 `any`/无标注混用（能标就标） |
| 无类型参数放前面 | 有类型参数放前面 |
| 浮点用 `/2.0` | 用 `float()` 转换 |
| 切片下限写 `0` | 写 `xs[:3]` |
| 字符串 `{` 写 `\{` | 直接写 `{` 当字面量 |
| 复杂逻辑拆小函数 | 一个 100 行大函数 |
| 文档写清返回语义 | 让调用方猜 nil/抛错 |

## 30. 错误处理与「结果约定」

Leash 当前**没有 `try/catch` / `Result` 类型**。那么库函数遇到错误怎么办？实践中有三种风格，
按推荐度排序：

### 30.1 返回 `nil` 表示「没有结果」

适合「查找类」函数：找不到就返回 `nil`，调用方用 `== nil` 判断。

```leash
fn mymap_get_or(m: map, k: str, default: str) -> str
    let v = json_get(m, k)
    if v == nil
        return default
    return v
```

### 30.2 返回「哨兵值」

适合「计算类」函数：约定某个特殊返回值代表错误，并在文档里写清。

```leash
// 返回 -1 表示除数为 0
fn safe_div(a: int, b: int) -> int
    if b == 0
        return -1
    return a / b
```

### 30.3 用 `out` 在主流程之前做「前置校验」

因为「能力」只在 main 里，真正的「报错并退出」应由 main 来做；库函数只负责返回错误信号。

```leash
// 库函数：返回 bool 表示「是否合法」
fn is_url(s: str) -> bool
    return str_starts(s, "http://") or str_starts(s, "https://")

// main：负责根据返回值决定打印什么
fn main
    let u = "ftp://x"
    if not is_url(u)
        out("非法 URL: " + u)
        return
    out("OK: " + u)
```

> 关键：**库函数不要自己 `out` 报错**（缺 io 能力）。把「是否出错」作为返回值，让 main 决定
> 怎么向用户展示、要不要 `exit()`（注意 `exit` 来自 `package os`，是 `exit(code)`）。

## 31. 字符串与文本处理须知

文本处理是 Leash 包的高频场景，这里集中讲几个要点：

### 31.1 字符串下标与切片

- `s[i]` 与 `s[i:i+1]`：取第 `i` 个字符（长度为 1 的字符串）。
- `s[a:b]`：取 `[a, b)` 子串；下限不能省（写 `0`）。
- `len(s)`：字符串长度（字符数，非字节数）。
- `ord(c)`：`c` 是长度为 1 的字符串，返回其码点整数。

### 31.2 字符串与数字不能直接比较

`"a" < "b"` 这种字典序比较**不支持**（坑位 6）。要比较请 `ord()` 转整数。

### 31.3 字符串拼接 `+`

`"a" + 1` 会得到 `"a1"`；`"x" + 3.5` 会得到 `"x3.500000"`。数字会自动转文本。反向不行（`1 + "a"` 会类型错）。

### 31.4 插值 vs 字面括号

`"你 {n} 岁"` 中 `{n}` 是插值。要输出字面 `{`，写 `\{`：

```leash
let tpl = "配置项: \{key} = value"
// 输出：配置项: {key} = value
```

### 31.5 多行文本用 `\n`

Leash 字符串用 `\n` 表示换行（如本教程 mdtool 示例）。`str_split(md, "\n")` 即可按行切分。

拿到一个陌生包（比如官方的 `csvx.ae`），如何快速上手？

## 33. 常见问题 FAQ

**Q1：我写了 `out(...)` 在库函数里，报「缺少能力: io」，怎么办？**
A：把 `out` 删掉，改成 `return 计算结果`，然后在调用方的 `fn main` 里 `out(...)`。这是坑位 2。

**Q2：`import "myutil"` 报「无法解析的模块」，怎么回事？**
A：按第 8 节排查：文件是不是 `lib/myutil.ae`？`LEASH_LIB` 指向对了吗？后缀是 `.ae` 吗？

**Q3：为什么我 `for ... in` 里写 `continue` 程序卡死？**
A：`continue` 在 `for ... in` 里会死循环（坑位 1）。改用 `if 条件 { ... }` 包住逻辑，或换 `while`。

**Q4：我想把整数变成浮点，写 `float(x)` 报未知函数？**
A：Leash 没有 `float()` 类型转换，用 `x / 2.0` 或 `x / 1.0`（坑位 0）。注意 `random` 包的 `float()` 是「随机浮点函数」，不要混用。

**Q5：map 的 key 能用表达式吗？`m[a[i]] = v` 报「无法解析的表达式」？**
A：赋值左值的 key 只能是简单变量或字符串字面量（坑位 7）。先 `let k = a[i]; m[k] = v`。

**Q6：为什么我的 JSON 字符串插值报「未定义变量」？**
A：字符串里的 `{` 被当插值了（坑位 5）。字面 `{` 写 `\{`；构造 JSON 优先用映射字面量 `{}`。

**Q7：切片 `xs[:3]` 报语法错误？**
A：下限不能省略（坑位 8），写 `xs[0:3]`；上限可省（`xs[2:]` 合法）。

**Q8：`c >= "0"` 报运算符错误？**
A：字符串不支持关系比较（坑位 6），用 `ord(c) >= 48`。

**Q9：`assert_eq` 失败为什么不停止？**
A：它只返回 `false`，不中断。需要「失败即停」请用 `if not assert_eq(...) { ... return }`（第 18.3 节）。

**Q10：map 遍历顺序每次不一样，正常吗？**
A：正常，当前实现不保证稳定（坑位 15）。需要有序请收集键到 list 再 `list_sort`。

**Q11：库函数能改调用方传进来的 list 吗？**
A：不能，Leash 是值语义（第 7.5 节）。请让函数返回新 list，由调用方接住。

**Q12：`package` 和 `import` 到底用哪个？**
A：纯 Leash 包用 `import "名称"`；需要宿主能力（文件/网络/时间/随机/AI…）用 `package 名称`（第 7 节）。

// numutil 包 v1.0.0
// 导入后可直接调用：numutil_is_even / numutil_clamp / numutil_sum_floats

import "math"

fn numutil_is_even(n: int) -> bool

fn numutil_clamp(n: int, lo: int, hi: int) -> int

fn numutil_sum_floats(xs: list) -> float

### 34.2 `textwrap`：按宽度折行（纯 Leash，依赖 string）

```leash
// textwrap 包 v1.0.0
// 导入后可直接调用：textwrap_wrap
// 依赖：string

import "string"

fn textwrap_wrap(s: str, width: int) -> list
    let words = str_split(s, " ")
    let mut lines = []
    let mut cur = ""
    let mut i = 0
    while i < len(words)            // 索引式 while 遍历 split 结果（坑位 19）
        let w = words[i]
        if len(w) > 0               // 跳过空词（坑位 20：循环内别 str_trim 切片）
            if len(cur) == 0
                cur = w
            else
                if len(cur) + 1 + len(w) <= width
                    cur = cur + " " + w
                else
                    lines = lines + [cur]
                    cur = w
        i = i + 1
    if len(cur) > 0
        lines = lines + [cur]
    return lines
```

### 34.3 `tablex`：渲染对齐表格（纯 Leash，依赖 string/list）

```leash
// tablex 包 v1.0.0
// 导入后可直接调用：tablex_render
// 依赖：string / list

import "string"
import "list"

fn tablex_render(rows: list) -> str
    if len(rows) == 0
        return ""
    let cols = len(rows[0])
    let mut widths = []
    let mut c = 0
    while c < cols
        let mut w = 0
        let mut r = 0
        while r < len(rows)
            let cell = rows[r][c]
            if len(cell) > w
                w = len(cell)
            r = r + 1
        widths = widths + [w]
        c = c + 1
    let mut out_lines = []
    let mut ri = 0
    while ri < len(rows)
        let mut cells = []
        let mut ci = 0
        while ci < cols
            let cell = rows[ri][ci]
            let pad = str_pad(cell, widths[ci], " ")
            cells = cells + [pad]
            ci = ci + 1
        out_lines = out_lines + [str_join(cells, " | ")]
        ri = ri + 1
    return str_join(out_lines, "\n")
```

### 34.4 `retry`：带重试的计时器（依赖 time + random）

```leash
// retry 包 v1.0.0
// 导入后可直接调用：retry_between
// 依赖：time / random（宿主能力）

package time
package random

// 返回每次尝试的耗时列表；调用方 main 负责打印
fn retry_between(times: int, secs: int) -> list
    let mut durations = []
    let mut i = 0
    while i < times
        let t0 = now()
        let _dummy = int(1, 100)   // 模拟一次随机工作
        let t1 = now()
        durations = durations + [(t1 - t0)]
        sleep(secs)
        i = i + 1
    return durations
```

### 34.5 `templ`：极简模板（演示 `\{` 转义）

```leash
// templ 包 v1.0.0
// 导入后可直接调用：templ_render
// 依赖：string / json

import "string"
import "json"

// 把 {name} 形式的占位替换成 data[name]；字面花括号用 \{ 表示
// 注意：这里为了演示「字符串里的 { 要转义」，模板里若含字面 { 必须写 \{
fn templ_render(tpl: str, data: map) -> str
    let keys = json_keys(data)
    let mut res = tpl
    let mut i = 0
    while i < len(keys)            // 索引式 while 遍历（坑位 19）
        let k = keys[i]
        // 占位符本身是 {name} 这种带花括号的字符串，必须用 \{ 和 \} 拼出来，
        // 否则单独的 "{" 会被当成「未闭合插值」而报编译错误（坑位 5）。
        let placeholder = "\{" + k + "\}"
        let v = json_get(data, k)
        res = str_replace(res, placeholder, v)
        i = i + 1
    return res

// 使用方（注意模板里的 \{name} 是「字面花括号」，不是插值）：
//     out(templ_render("hi \{name}", {"name": "Leash"}))   // 输出：hi Leash
// 若写成 "hi {name}"（不转义），Leash 会把 {name} 当作插值变量去求值，报「未定义变量 name」。
```

> `templ_render` 用 `str_replace` 做占位替换，不涉及「字符串字面量里写 `{`」的问题（模板来自变量）。
> 只有当你**在源码字符串里硬写** `{` 时才需要 `\{`（坑位 5）。

## 35. 用 stub 隔离宿主依赖做测试

有些包依赖宿主能力（如 `file`/`http`），单测时不想真的读文件或联网。怎么办？

**思路**：把「宿主动作」封装成一个小函数，测试时用一个「纯 Leash 版」替换它。因为 Leash 按名字解析，
你在测试文件里**先定义**一个同名函数，再 `import` 被测包，就能「覆盖」掉宿主版。

```leash
// tests/test_mycache_stub.ae
// 先定义 stub（覆盖 file 的原生 read/write），再 import 被测包
fn read(path: str) -> str
    // 注意：Leash 字符串字面量无法直接写双引号（" 会结束字符串），
    // 也不能直接写 { }（会被当插值）。所以这里用 stringify 生成带引号的 JSON 文本，
    // 等价于「从文件读到的 {\"user\":\"stub\"}」。（坑位 5）
    return stringify({"user": "stub"})

fn write(path: str, content: str)
    // 什么也不做，纯测试

import "simplecache"
import "assert"

fn main
    // 这里 simplecache 用的是上面 stub 的 read/write，不会碰真实文件
    let v = simplecache_get("/tmp/x", "user")
    assert_eq(v, "stub", "stub 读取生效")
    out("stub 测试通过")
```

> ⚠️ 注意：这种「覆盖」依赖 `assemble()` 的收集顺序，属于**测试技巧**而非语言特性。
> 生产代码不要依赖它；它只用于隔离宿主做单测。

package os             // 得 env(name) / cwd() / exit()
package re             // 得 ismatch / find（避免 {n}）
package http           // 得 get(url) / post(url, body)

// —— 在 main 里打印，库函数只返回 ——
fn main
    out(mytool_double(21))

// —— 避开这些坑 ——
// 0. 没有 float() 转换；整数除法截断，浮点用 /2.0
// 1. for ... in 里不要 continue（用 if 包逻辑）
// 2. 库函数不要 out（缺 io 能力）
// 3. 变量名别用 out/in/match/cap/int/float/str/bool/list/map
// 4. 字符串里的 { 要写成 \{
// 5. 字符串不能 >= 比较，用 ord()
// 6. m[a[i]] = v 不行，先 let k = a[i]; m[k] = v
// 7. 切片下限不能省：xs[:3] -> xs[0:3]
// 8. 没有三元 ?:，用 if/else
// 9. 参数：无类型在前，有类型在后；map 参数标 : map
// 10. 别定义 get/post/now/sleep 等原生占用名
// 11. 别假设 map 遍历顺序稳定
// 12. json_get 取不到返回 nil，用 == nil 判断
```

## 39. 附录 C：术语表

| 术语 | 含义 |
|------|------|
| 纯 Leash 包 | 用 Leash 自身写的 `.ae` 包，用 `import "名称"` 引入 |
| 原生包 | 用 C++ 实现、由宿主注入的包，用 `package 名称` 引入 |
| 能力（capability） | 宿主授予的权限（如 io、file、network）。`out`/`in` 是 io 能力 |
| 全局命名空间 | 所有 `import`/`package` 的函数最终合并进同一张函数表；重名会静默覆盖 |
| 值语义 | list/map 作为参数传递时是副本，函数内修改不影响外部 |
| 前缀纪律 | 所有导出函数以 `包名_` 开头，避免全局冲突 |
| 燃料（fuel） | VM 的执行步数上限；过深递归或死循环会触发「燃料耗尽」 |
| 插值 | 字符串里 `{expr}` 会被求值替换；字面 `{` 需写 `\{` |
| 传递依赖 | import A 时，A 依赖的 B 的函数也会进入全局命名空间 |
| 断言包 | `assert`/`assertx`，只比较返回 bool，不中断程序 |
